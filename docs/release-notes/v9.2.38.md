# BOSS v9.2.38

**Release Date:** 2026-07-09

Fixes plugin self-reloads: `PluginLoaderDelegate.reloadPlugin` now runs its unload+load orchestration on a detached, delegate-owned scope, so a plugin that reloads its own plugin (Toolbox self-update, evolver hot-reload) is no longer force-unloaded and left stranded when its scope is cancelled mid-reload — unlocking restart-free Toolbox self-updates (#849).

## Highlights

- **Plugin self-reloads survive the caller's teardown**: `reloadPlugin` previously ran on the *caller's* coroutine, so reloading your own plugin cancelled the scope between unload and load — leaving the plugin unloaded and never reloaded (why Toolbox self-updates still needed a restart). Mirroring the #848 `hotSwapApiLayer` pattern, the unload+load now runs as a detached job on a delegate-owned scope; a caller torn down by its own reload gets the cancellation at `await` while the job completes, with an `invokeOnCompletion` handler logging any unobserved throw. `resetPluginInstances` inherits the protection (#849).
- **Coalesced, concurrency-guarded reloads**: A new `KeyedDetachedJobs` extracts the detach-and-coalesce pattern the detached reloads lacked — concurrent `reloadPlugin` calls for one `pluginId` join the in-flight job instead of interleaving two uninstall+load sequences, with identity-checked removal so a finished job's cleanup can't evict a newer one. `resetPluginInstances` now rethrows `CancellationException` before its general catch, so the self-reset case no longer surfaces a spurious "Exception resetting plugin instances" error and a wrong `0` return (#849).

## Bug Fixes

- **plugin-platform**: run delegate plugin reloads in a detached scope — `PluginLoaderDelegate.reloadPlugin` orchestrates unload+load as a detached job on a delegate-owned scope (never cancelled; in-flight reloads must outlive their initiator) instead of on the caller's coroutine, so a plugin reloading its own plugin is no longer force-unloaded and stranded when its scope cancels between unload and load; concurrent reloads for a given `pluginId` coalesce via the new `KeyedDetachedJobs`, `resetPluginInstances` rethrows `CancellationException` before its general catch, and a `NoClassDefFoundError` from a closed classloader reaches the `onDetachedFailure` hook rather than vanishing with a torn-down caller — unlocking restart-free Toolbox self-updates (#849)
