# BOSS v9.2.15

**Release Date:** 2026-06-27

Adds a `secret.read` RBAC permission so the Secret Manager can move off the legacy `requiresAdmin` flag onto the granular permission model, restyles plugin toasts to match BOSS dialogs, and pulls in a batch of toolchain dependency bumps (Kotlin 2.4.0, Compose Multiplatform 1.11.1).

## Highlights

- **`secret.read` permission for Secret Manager**: New granular RBAC permission, granted to `admin` + `boss_admin`, lets the Secret Manager panel be gated via `requiredPermissions` instead of the admin-only `requiresAdmin` flag (#794).
- **Plugin toasts match BOSS dialogs**: Plugin sandbox toast notifications are restyled to align with the app's dialog look and feel (#795).
- **Toolchain bumps**: Kotlin 2.4.0 and Compose Multiplatform 1.11.1, plus JNA, Logback, and JUnit updates (#796).

## Features

- **rbac**: Add `secret.read` permission for Secret Manager gating — creates the permission and grants it to `admin` + `boss_admin` so the per-user secret vault can adopt the `requiredPermissions: ["secret.read"]` model (panel visibility only; RPCs remain scoped to `auth.uid()`). Idempotent migration `20260628000000_secret_read_permission.sql` (#794)

## Improvements

- **plugin**: Style plugin sandbox toasts (`PluginToastHost`) to match BOSS dialogs (#795)

## Dependencies

- Bump Kotlin from 2.3.21 to 2.4.0 (#796)
- Bump Compose Multiplatform from 1.11.0 to 1.11.1 (#796)
- Bump JNA from 5.18.1 to 5.19.1 (#796)
- Bump Logback (`logback-classic`) from 1.5.32 to 1.5.34 (#796)
- Bump JUnit Jupiter from 6.0.3 to 6.1.0 (#796)
