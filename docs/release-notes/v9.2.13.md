# BOSS v9.2.13

**Release Date:** 2026-06-27

Introduces a granular RBAC role hierarchy with permission-based plugin gating and a host window-resize API that lets embedded BossTerm panes fit the host OS window, alongside a fix making the email-link auth redirect publicly reachable.

## Highlights

- **Granular role hierarchy + permission-based plugin gating** (#793): A new `role_hierarchy` (a parent role inherits its descendants' permissions) drives permission inheritance, plugin visibility, and grant delegation. Effective permissions ride the `user_permissions` JWT claim (magic-link and passkey), role-management RPCs are re-gated from the literal admin role to `authorize('role.<verb>')` so domain admins work, delegation is tree-derived (you can only grant strictly below yourself), and plugins now declare `requiredPermissions` that gate whether they register at all.
- **Window fit/restore API for embedded panes** (#779): `WindowManager.fitWindowByDelta()` / `restoreWindowSize()` let BossTerm's "Fit host to my screen" resize the host OS window even when the terminal is embedded as a BossConsole pane — HiDPI-aware (physical-px → logical-dp), screen-clamped on multi-monitor, with pre-fit size/placement restore. Reached via the terminal-tab plugin's existing reflection pattern, so it ships as a no-op until called.

## Features

- **rbac**: Role hierarchy with cycle-safe `get_role_descendants`/`get_effective_permissions`, `authorize()` rewritten to walk the descendant closure, effective permissions injected into the `user_permissions` JWT claim via `custom_access_token_hook`, tree-derived grant delegation (`get_grantable_roles`, gated assign/remove), and host-side `PluginManifest.requiredPermissions` so `DynamicPluginManager` only registers a plugin when the user's effective permissions cover it. (#793)
- **window**: `WindowManager` fit/restore window-resize API — `fitWindowByDelta(windowId, dW, dH)` (incremental, screen-clamped, HiDPI px→dp, drops a Maximized MAIN window to Floating) and `restoreWindowSize(windowId)`, plumbed through `BossWindowState`/`BossWindow` via a buffered `WindowSizeRequest` event stream with `replay = 1` so early-startup fits aren't dropped. (#779)

## Bug Fixes

- **redirect**: Set `verify_jwt=false` on the email-link redirect edge function so the public, unauthenticated email verification link can reach it. (#787, #791)

## Improvements

- **rbac**: Security-review hardening — gate the privileged "read all users" RLS on `role.read` (not the baseline `user`-carried `user.read`, which would have exposed the full user list to every authenticated user), plus helper-exposure, anonymous-grant, and seed-drift fixes. (#793)
