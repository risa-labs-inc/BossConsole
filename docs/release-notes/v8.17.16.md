# BOSS v8.17.16

**Release Date:** 2026-03-07

Smart file routing and plugin crash recovery release. Media files (images, PDFs, video, audio) now automatically open in the browser tab while code files route to the editor, with a new "Open With" context menu for manual override. Four plugin stability bugs are also fixed.

## Highlights

- **Smart file routing**: Images, PDFs, video, and audio open in the browser tab automatically; code files route to the editor
- **"Open With" context menu**: Nested submenu lets you explicitly open any file in the Terminal, Browser, or Editor
- **Plugin crash recovery**: DISABLED plugins no longer trigger repeated crashes on every recompose; side panel crash UI now renders correctly
- **Windows path fix**: File URLs now use `File.toURI()` for correct handling of backslashes, drive letters, and special characters

## Features

- **tabs**: Add `initialCommand` to `TabsComponent.addTerminalTab` — passes the file path to the terminal when opening via "Open With > Terminal"
- **panel**: Smart file routing in `SplitView` — routes images, PDFs, video, and audio to browser tab; adds `openFileInBrowser` and `openFileInEditor` to `SplitViewOperations` API; adds `subMenu` support to `ContextMenuItemData` for nested context menus

## Bug Fixes

- **panel**: Fix `openFileInEditorTab` returning no-op when the file was already open in a browser tab — now falls through to create a new editor tab when only a browser tab exists
- **panel**: Fix crash prompt not showing for side panel plugin crashes — add crash state from `PluginCrashRegistry` to `key()` in `SidePanel` to force subtree teardown and rebuild on crash
- **panel**: Block rendering of DISABLED plugins in `PluginErrorBoundary` — prevents repeated `NoSuchMethodError` crashes from re-rendering disabled plugins on every composition
- **panel**: Fix Windows file URL construction using `File(path).toURI().toString()` instead of raw `"file://$path"` string interpolation; change synthesized error for DISABLED plugins from `NoSuchMethodError` to `RuntimeException`; replace `Triple` with `PanelTabMatch` data class for `findPanelWithEditorTab` return type
