# BOSS v8.17.11

**Release Date:** 2026-03-06

Bug fix release resolving macOS default browser detection failures and a null pointer crash in the browser handler check.

## Highlights

- **Reliable macOS Default Browser Detection**: Replaces fragile regex-based plist parsing with `LSCopyDefaultHandlerForURLScheme` via Swift script for accurate, OS-supported browser detection
- **NPE Fix in Default Browser Check**: `isDefaultBrowser()` no longer crashes when `getDefaultHandlerForScheme()` returns null — comparison is now null-safe
- **Platform-Aware Instructions Dialog**: The "Set as Default Browser" instructions dialog now shows the correct steps for macOS vs. Windows instead of always displaying Windows text
- **System Settings URL Updated**: The macOS fallback deep link now targets the correct pane for macOS Ventura and later

## Bug Fixes

- **browser**: Replace `getDefaultHandlerViaDefaults()` regex/plist approach with `LSCopyDefaultHandlerForURLScheme` Swift invocation for reliable default browser detection on macOS
- **browser**: Fix NPE in `isDefaultBrowser()` by calling `BUNDLE_ID.equals(httpDefault)` instead of `httpDefault.equals(BUNDLE_ID)`, safely handling null returns from `getDefaultHandlerForScheme()`
- **browser**: Update System Settings fallback URL to use the correct pane identifier for macOS Ventura+
- **browser**: Make "Set as Default Browser" instructions dialog platform-aware (macOS vs. Windows)
- **browser**: Remove dead `registerURLHandler()` code and stale "Fluck Browser" comment
