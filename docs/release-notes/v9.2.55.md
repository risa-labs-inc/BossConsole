# BOSS v9.2.55

**Release Date:** 2026-07-23

Fixes an in-app updater failure on macOS App Translocation and hardens the public CI pipeline against leaking signing secrets, alongside a large documentation refresh (README showcase, "How BOSS compares", shared agent guidance).

## Highlights

- **Updater survives macOS App Translocation**: updates now correctly restart the app even when Gatekeeper runs BOSS from a randomized translocated path (#16).
- **CI secret leak closed**: DigiCert signing secrets can no longer leak into public CI logs (#6).
- **README & comparison overhaul**: the README is rewritten as a showcase (AI agents, governance, Toolbox, BossTerm sharing) and gains a "How BOSS compares" open-source capability matrix (#8, #11, #12).
- **boss-plugin-api → 1.0.68**: dependency pin bumped (#17).

## Bug Fixes

- **updater**: survive macOS App Translocation so updates restart the app (#16)

## Improvements

- **security**: stop DigiCert signing secrets leaking into public CI logs (#6)
- **ci**: commit automated version bumps as a human identity, not a bot (#9)
- **docs**: rewrite README as a showcase — AI agents, governance, Toolbox, BossTerm sharing (#8)
- **docs**: document Toolbox and MCP in README; fix open-source staleness (#7)
- **docs**: add a centered hero to the README opening (#10)
- **docs**: add "How BOSS compares" — open-source alternative + capability matrix (#11)
- **docs**: reopen + expand comparison (first open-source harness, evolvable toolbox, OSS ecosystem) (#12)
- **docs**: pull "How BOSS compares" back for revision; finish Fluck rebrand
- **docs**: link every Toolbox plugin to its repo (#13)
- **docs(compare)**: add Runtime (JVM multi-threaded) and hot-reload rows (#14)
- **docs**: share agent guidance across Codex and Claude (#15)

## Dependencies

- Bump `boss-plugin-api` pin to 1.0.68 (#17)
