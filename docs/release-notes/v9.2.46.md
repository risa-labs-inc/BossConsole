# BOSS v9.2.46

**Release Date:** 2026-07-16

Gives the browser home page a stable identity so back-navigating to it shows a Home icon and title instead of a blank tab, and refreshes four backend dependencies.

## Highlights

- **Home-page tab identity**: The browser home page (empty/`about:blank` URL, which renders the dashboard) now has an identity of its own — `FluckTabInfo` shows a Home icon while a tab is on the home page (no more stale favicon from the previous page), and landing on home sets the title to "Home". Blank titles are ignored so a tab chip is never blanked, and the fix holds even for older fluck-browser plugin versions (#865).

## Bug Fixes

- **browser**: home-page tab identity — Home icon/title, never blank a tab title. `FluckTabInfo.isHomeUrl`/`HOME_TITLE` is now the single definition of the home state (used by both the tab icon logic and `BossTabUpdateProvider.updateUrl`, previously duplicated and prone to drift); a home landing records "Home" in navigation history and clears the favicon cache key, and `updateTitle` ignores blank titles. Adds `FluckTabInfoTest` covering `isHomeUrl` and the exact home-landing transform (title, favicon, current URL, icon, history entry) (#865).

## Dependencies

- **deps**: Bump kotlinx-serialization from 1.9.0 to 1.11.0 (#839)
- **deps**: Bump gradle-wrapper from 9.5.1 to 9.6.1 (#843)
- **deps**: Bump ch.qos.logback:logback-classic (#867)
- **deps**: Bump io.grpc:protoc-gen-grpc-java (#868)
- **deps**: Bump grpc from 1.82.1 to 1.82.2 (#866)
