# BOSS v9.2.31

**Release Date:** 2026-07-04

Feature release adding native macOS platform passkeys (iCloud Keychain) for in-browser WebAuthn, a Supabase-first browser-engine download channel with a version picker in Settings, and a `boss://split` MCP deep link — plus the macOS build fix that unblocked this release.

## Highlights

- **Native macOS platform passkeys (iCloud Keychain)**: JxBrowser's embedded Chromium exposes no platform authenticator on macOS, so `navigator.credentials.get/create` never prompted for a platform passkey. BOSS now routes those ceremonies to Apple's browser-passkey APIs so iCloud Keychain passkeys work like they do in Chrome/Safari — behind an origin-binding + Public-Suffix-List `rpId` guard, top-frame-only injection, and a runtime provisioning probe that keeps the whole feature a strict no-op (clean Chromium fallback) until Apple provisions the App ID (#823).
- **Supabase-first browser-engine downloads + version picker**: Browser-engine archives now resolve through Supabase `app_releases` (`boss-chromium`) first with GitHub as backup, mirroring the app updater, with sha256 verification before extraction. A new **Browser Engine** settings section shows installed/default versions and lets you pin a published one; installs stage into `~/.boss/boss-chromium.pending` and swap in at startup (via a `.old` backup) so a running engine is never replaced in place (#832).
- **`boss://split` deep link**: An MCP-driven agent can now split the main window (the same action as View → Split Vertically/Horizontally) via `boss://split?orientation=vertical|horizontal`, with reliable window resolution so it no longer silently no-ops when BOSS lacks OS focus (#834).
- **macOS build unblocked**: A literal `--` inside an XML comment in `BOSS.entitlements` (added with the passkey work) broke every macOS `codesign` step with an AMFI parse error; the comment was reworded with no change to the entitlements themselves (#835).

## Features

- **fluck**: Native macOS platform passkeys (iCloud Keychain) for in-browser WebAuthn via Apple's `ASAuthorizationWebBrowserPublicKeyCredentialManager`, compiled to a universal in-process `libBossWebAuthn.dylib`; includes origin binding to the engine's real top-frame URL, real PSL-based `rpId` validation, top-frame/popup-aware injection, ES256-only routing with USB fallback, `excludeCredentials`→`InvalidStateError` mapping, and denial recovery that flips back to Chromium fallback for the session (#823)
- **engine**: Supabase-first browser-engine download source (`ChromiumReleaseSource`, `boss-chromium`) with GitHub backup and sha256 verification, plus a **Browser Engine** settings section to view versions and pin a published engine, with staged installs promoted safely at startup (#832)
- **mcp**: `boss://split?orientation=vertical|horizontal` deep link to split the main window from an MCP-driven agent (#834)

## Bug Fixes

- **macos**: Remove the invalid literal `--` from inside the `BOSS.entitlements` XML comment that broke every macOS `createDistributable`/`codesign` step with an `AMFIUnserializeXML` syntax error (#835)
- **engine**: The setup download window now grows in its error state and bounds the error-message height so the Retry/Exit buttons are no longer clipped (#832)
- **update**: The update dialog's "What's new" now shows the curated `docs/release-notes/vX.Y.Z.md` notes instead of the boilerplate auto-generated GitHub body — `sync-release.yml` prefers the curated file, `release-notes.yml` PATCHes them into the Supabase `app_releases` row, and a backfill script repairs already-published versions (#831)

## Improvements

- **docs(claude)**: Relaxed the absolute "do NOT run `./gradlew run`" rule to permit launching the app in a dedicated bottom split pane, and documented both terminal MCP servers a session may host (`mcp__boss__*` and `mcp__bossterm__*`) (#833)
