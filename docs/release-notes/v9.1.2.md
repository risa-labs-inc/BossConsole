# BOSS v9.1.2

**Release Date:** 2026-05-08

Major release extracting the microkernel runtime out of the BOSS tree and distributing it as a separately versioned plugin via GitHub releases, with an IPC version compatibility gate to prevent mismatches. Also bundles a JxBrowser 9.0.2 upgrade with branding-patch fixes, a new BOSS Lite release workflow, and a sweep of major dependency updates (Kotlin 2.3.21, gRPC 1.81.0, protobuf 4.34.1, Gradle 9.5.0).

## Highlights

- **Microkernel runtime distributed as a plugin**: The in-tree `boss-microkernel-runtime` module has been removed and the runtime is now published as a versioned plugin to a GitHub release, fetched on demand. An IPC version compat gate refuses to load runtimes whose protocol does not match BOSS, with a clear error if a user tries to install the runtime via the plugin store.
- **JxBrowser 9.0.2 + branding patch fixes**: Default JxBrowser bumped to 9.0.2; the Chromium-Branding tool now patches out the `(Plugin)` helper string that previously caused a branding-patch panic.
- **BOSS Lite release workflow**: New signed release pipeline for BOSS Lite reusing BossConsole signing secrets, with a hardened macOS keychain setup (`.keychain-db`, login keychain in search list, no premature EXIT trap) and concurrency guards.
- **Major dependency sweep**: Kotlin 2.3.10 → 2.3.21, gRPC 1.72.0 → 1.81.0, protobuf 4.31.1 → 4.34.1, Decompose 3.4.0 → 3.5.0, Gradle wrapper 9.3.1 → 9.5.0, plus `gradle/actions` v3 → v6 and `softprops/action-gh-release` v2 → v3.

## Features

- **kernel**: Phase 0 IPC version compatibility gate for the microkernel runtime — BOSS refuses to load a runtime JAR whose IPC protocol version does not match
- **kernel**: Phase 1 — aggregate and publish upstream IPC JARs from BossConsole CI so external runtime builds can depend on them
- **kernel**: Distribute the microkernel runtime as a plugin via GitHub releases instead of bundling it in the BOSS tree
- **release**: Add BOSS Lite release workflow using BossConsole signing secrets, including macOS notarization and code signing

## Bug Fixes

- **browser**: Fix branding-patch panic by patching out the `(Plugin)` helper string from the Chromium-Branding tool
- **browser**: Bump JxBrowser default to 9.0.2 to pick up upstream fixes alongside the branding patch update
- **kernel**: Switch `PluginStoreSetup` to reflection-only access for `KernelBootstrap` and `IpcVersion` so monolith builds load cleanly even when kernel modules are absent
- **kernel**: Throw a clear error when a user attempts to install the microkernel runtime from the plugin store (it is bootstrap-only, not a regular plugin)
- **kernel**: Guard runtime JAR identification by `pluginId` rather than only the Gradle filename prefix
- **kernel**: Quiet stale-manifest log and guard the plugin-install path against accidentally treating a runtime JAR as a regular plugin
- **kernel**: Fix template group ID and authenticate GitHub API calls in the runtime distribution flow
- **kernel**: Fix microkernel runtime plugin review concerns (resource leak, ZIP64 handling, manifest version, async update, oversize `/github` payloads)
- **release**: Surface notarytool errors in the release workflow so signing/notarization failures fail loudly instead of silently
- **release**: Fix macOS keychain setup — use `.keychain-db`, append `login.keychain-db` to the search list
- **release**: Fix Lite keychain being deleted before the build — remove the premature EXIT trap from the setup step (#694)

## Improvements

- **kernel**: Address security review — atomic manifest update, dedup, size cap, and resilient `isKernelMode` detection
- **kernel**: Address security review — fix resource leak, handle ZIP64, validate manifest version, async update, reject oversized `/github` responses
- **release**: Address PR review on Lite workflow — add concurrency guard, keychain cleanup trap, and `app.lite.build` fallback

## Security

- **kernel**: Compute SHA-256 server-side and restrict external JAR host to prevent client-supplied hash bypass
- **kernel**: Close SHA-256 bypass and manifest spoofing in `/github/metadata` endpoint

## Cleanup

- **kernel**: Phase 4 — delete the in-tree `boss-microkernel-runtime` module now that it is distributed as a plugin
