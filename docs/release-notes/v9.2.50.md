# BOSS v9.2.50

**Release Date:** 2026-07-20

Threads a `showHidden` opt-in through the read-side file scanners so plugins can list dot-entries on demand, and tidies the repo layout by moving the microkernel modules under `modules/` and renaming `plugins/` â†’ `plugin-platform/` to clarify it's the host-side plugin SDK, not the shippable plugins.

## Highlights

- **`showHidden` on read-side file scans**: `FileSystemDataProvider` now advertises `supportsHiddenEntries = true` and threads a `showHidden` flag through `scanDirectory` / `scanDirectoryWithDepth` / `directoryHasChildren`, so callers can opt into dot-entries. Works end-to-end for out-of-process plugins (the `FileSystemService` proto has carried `include_hidden` since v1), the visibility filter is hoisted to `commonMain` so desktop/android can't drift, and `build`/`node_modules` stay skipped regardless. Legacy methods are unchanged (dot-entries still filtered), and it lets the Codebase plugin retire its duplicated scanner. Requires boss-plugin-api 1.0.66 (#881).
- **Repo layout cleanup**: The 13 `boss-*` microkernel Gradle modules move under `modules/` and `plugins/` is renamed to `plugin-platform/`, both purely cosmetic â€” flat Gradle paths (`:boss-ipc`, â€¦) and type-safe accessors are preserved, so no behavior changes (#879, #880).

## Features

- **files**: Honor `showHidden` on read-side file scans. `scanDirectory` / `scanDirectoryWithDepth` / `directoryHasChildren` overloads thread the flag through the platform scanners (desktop and android fully; ios/wasm mocks delegate), the IPC `FileSystemDataProviderProxy` honors it against the kernel-side service, and the visibility filter (`isVisibleScanEntry` + `scannerSkippedDirectoryNames`) is shared from `commonMain`. Adds scanner tests. Requires boss-plugin-api 1.0.66 (#881).

## Improvements

- **modules**: Move the microkernel modules under `modules/` to declutter the repo root. The `boss-ipc`, `boss-service-*`, `boss-orchestrator`, `boss-ui-sdk`, `boss-mastery-*`, and `boss-app-*` modules physically relocate into `modules/` while keeping their flat Gradle paths via `projectDir` overrides in `settings.gradle.kts` â€” every `project(":boss-ipc")` reference and `projects.*` accessor is unchanged, and only `settings.gradle.kts` plus one `upstream-artifacts.gradle.kts` path needed edits (108 files, 0 content change) (#879).
- **plugin-platform**: Rename `plugins/` â†’ `plugin-platform/` to clarify it's the host-side plugin platform/SDK â€” the infrastructure that loads and runs plugins, not the shippable plugins (those live in the separate `boss_plugins` repo). Renames the container and its Gradle coordinate `:plugins:*` â†’ `:plugin-platform:*` across `settings.gradle.kts`, all `.kts` accessors, the Maven Central publish workflow, docs, and scaffolding (20 modules) (#880).
