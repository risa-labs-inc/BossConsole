# BOSS v9.2.48

**Release Date:** 2026-07-19

Removes the never-used plugin dependency resolver (~590 net lines) and hardens `SemanticVersion` — now the sole version-comparison primitive gating plugin updates — with correct SemVer precedence and stricter parsing.

## Highlights

- **Dead plugin-dependency-resolver removed**: BOSS expresses cross-plugin needs via soft runtime lookup (`context.getPluginAPI(X::class.java): T?`, null-handled), not the manifest `dependencies` field — no plugin declares any dependency, so `PluginDependencyResolver` (hard, version-ranged, resolve-before-load) had zero input and zero callers. It and its result types, `VersionRange`/`VersionConstraint`, `PluginDependencyException`, and composeApp's unused module edge are gone; the manifest `dependencies` field (still read by `DynamicPluginManager` for uninstall-safety) and `SemanticVersion` are kept (#876).
- **`SemanticVersion` correctness fixes**: With the resolver gone, `SemanticVersion` is the only primitive gating plugin updates. It now implements SemVer §11 prerelease precedence identifier-wise (fixing a lexical bug where `rc.10` sorted below `rc.9`, which flows into `PluginUpdateManager.isNewerVersion`), drops the `build` field that broke the `Comparable` contract (`1.0.0+a` vs `1.0.0+b` gave `compareTo==0` but `equals==false`), and rejects malformed inputs (`1..2`, `1.x`, negatives, empty prerelease/build sections) — covered by 12 new unit tests (#876).

## Improvements

- **chore**: Remove the unused plugin dependency resolver (~590 lines) — deletes `PluginDependencyResolver` and its `DependencyResolutionResult`/`MissingDependency`/`DependencyConflict`/`DependencyNode`/`DependencyTreeNode` types, the resolver-only `VersionRange`/`VersionConstraint`, the zero-thrower `PluginDependencyException`, and composeApp's dead `implementation(pluginDependency)` edge. The `plugin-dependency` module is retained as `SemanticVersion` only, hardened to proper SemVer §11 precedence and stricter parsing, with its now-dead deps (coroutines, serialization, plugin-api-core, plugin-logging, the serialization plugin) trimmed since it is pure Kotlin (#876).
