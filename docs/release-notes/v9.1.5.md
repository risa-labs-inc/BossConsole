# BOSS v9.1.5

**Release Date:** 2026-05-12

Follow-up bug-fix for the v9.1.4 form-submit popup work: concurrent POST stashes to the same URL (e.g. clicking Print twice in OncoEMR before the first new tab finishes constructing) are now queued FIFO and stash/consume mutate atomically, so the first tab no longer replays the wrong body or falls back to an empty GET.

## Highlights

- **Double-print regression fixed**: Back-to-back form-submit popups to the same URL are dequeued in submission order instead of the second overwriting the first.
- **Atomic stash/consume**: Per-URL queue mutations and map membership are now wrapped in `ConcurrentHashMap.compute`, closing a TOCTOU window where a racing stash could be silently dropped.

## Bug Fixes

- **browser**: Queue concurrent POST stashes to fix the OncoEMR double-print regression — `BrowserServiceImpl` replaces the per-URL single entry with a FIFO `ConcurrentLinkedDeque`, drains stale heads under TTL, and wraps stash/consume/GC in `ConcurrentHashMap.compute` so a racing `stashPopupPost` serializes on the key instead of clobbering (or being clobbered by) `consumePopupPost`. Also extracts `dropStaleHeads` and trims the duplicate `ConcurrentLinkedDeque` qualification. (#707)
