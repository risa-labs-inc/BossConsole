# BOSS v9.1.11

**Release Date:** 2026-05-22

First three phases of migrating the terminal plugin from in-process to out-of-process so `bossterm` can ship updates independently of BossConsole — additive IPC contract (`boss-ipc` 1.0.0 → 1.1.0), host-side Compose grid renderer behind the `boss.terminal.oopRenderer` flag (default off), and gRPC connection scaffolding ready for Phase D wiring.

## Highlights

- **Terminal OOP IPC contract (Phase A)**: Extends `services/terminal.proto` with server streams (`StreamGrid` / `StreamCursor` / `StreamScrollback` / `StreamShellEvents`) and unary RPCs (`SendKeyEvent` / `SendComposition` / `SendMouseEvent` / `SetTheme`). New messages cover cell-grid deltas with grapheme cluster offsets, packed `CellAttr` style bitmasks, cursor shape + blink hint, windowed scrollback, OSC-133 shell events, modifier-aware key encoding, IME composition, mouse events, and a host-controlled theme palette. `IpcVersion.CURRENT` bumped to 1.1.0; plugins with `minIpcVersion=1.0.0` stay compatible via the existing semver gate. (#733)
- **Host-side Compose grid renderer (Phase B)**: `TerminalGridRenderer` is a pure-Compose composable — single `Canvas` with `TextMeasurer.drawText`, monospace metrics, cursor overlay (block / bar / underline), and no `bossterm` imports. Backed by `GridState` (pure-logic mirror of the cell grid with style flyweight interning), `ThemePalette` (proto-style → Compose mapping), and `KeyEventEncoder` (Compose `KeyEvent` → `SendKeyEventRequest`). Hidden behind `boss.terminal.oopRenderer` flag (env → sysprop → local.properties via existing `ConfigLoader`), default off — no behaviour change in the default build. (#733)
- **gRPC connection scaffolding (Phase C)**: `TerminalGridConnection` owns a `ManagedChannel` + `TerminalServiceCoroutineStub` + `TerminalGridClient` with `AutoCloseable` lifecycle. Two factories — `connect(address, sessionId)` opens and owns the channel; `adopt(channel, sessionId)` wraps an externally-owned channel from `OutOfProcessPluginSpawnerImpl.getChannel()` (Phase D wiring point). `LoopbackTerminalServer` provides an in-JVM `TerminalServiceCoroutineImplBase` over real gRPC + Unix domain sockets for dev tooling and integration tests. (#733)
- **Cell-run coalescing in the renderer**: `TerminalGridRenderer.drawCells` coalesces contiguous same-style cells into runs and emits one `drawText` per run instead of one per cell. 80×24 worst-case drops from 1920 measure calls per frame to a handful in steady state, matching the proto's `CellRun` intent. (#733)

## Features

- **terminal**: Terminal OOP renderer — Phase A (IPC proto) + Phase B (host renderer) + Phase C (connection scaffolding). Pure-Compose `TerminalGridRenderer` backed by `GridState` / `ThemePalette` / `KeyEventEncoder`, wired through `TerminalGridConnection` to a `TerminalServiceCoroutineStub` over gRPC + Unix domain sockets. `StubTerminalGridSource` exercises the renderer end-to-end with canned data + key echo; `TerminalRenderDevWindow` opens a standalone dev preview when the flag is on. `OutOfProcessPluginSpawnerImpl`, plugin manifests, and existing terminal paths are untouched — the new factories sit ready for Phase D to call. (#733)

## Bug Fixes

- **terminal**: `GridState.applyDelta` now requires `rev == previous + 1` for incremental frames, never lowers the watermark, and rejects incrementals that arrive before any full redraw. The earlier logic silently accepted duplicates and out-of-order frames and then masked subsequent real gaps; rejections now log via `BossLogger.trace` under `LogCategory.TERMINAL`. (#733)
- **terminal**: `KeyEventEncoder` only attaches `text` to `KeyDown` so `KeyUp` no longer echoes the same character through IPC. `KeyUp` still carries `keyCode` and modifiers so chord-release detection remains possible. (#733)
- **terminal**: `LoopbackTerminalServer.keyEventCount` switched to `AtomicInteger` — `@Volatile` only guarantees visibility, not read-modify-write atomicity under concurrent `sendKey` calls. (#733)
- **terminal**: `StubTerminalGridSource` synchronizes the `keystrokes` `StringBuilder` so the emitter coroutine and `sendKey` callers can't race on mutation/read. (#733)
- **terminal**: `TerminalGridRenderer` now renders the block cursor as a filled rect with `cursorBackground` rather than a hollow stroke at 0.4 alpha — the latter looked unfocused even when the terminal was focused. (#733)
- **build**: Bump `:boss-ipc` Gradle module version to 1.1.0 to match `IpcVersion.CURRENT`. The earlier Phase A commit only updated the Kotlin constant, leaving the module at 1.0.0 — which broke `:assembleUpstreamJars` and downstream consumers (`boss-microkernel-runtime`, plugin JARs) that resolve `boss-ipc` by filename like `boss-ipc-1.1.0.jar`. Both should track the IPC wire-format version, not BossConsole's app version. (#733)

## Improvements

- **terminal**: `TerminalGridConnection.connect` keeps a reference to the `BossIpcClient` it constructs and delegates `close()` to its `shutdown()` — eliminates the "wrapper dropped after extracting channel" smell from the original Phase C wiring. (#733)
- **terminal**: `KeyEventEncoder` extracts a pure-data `encodeRaw(...)` that the Compose-typed overload delegates to. Compose Multiplatform 1.11.0 changed the desktop `KeyEvent` constructor to wrap `InternalKeyEvent` rather than AWT `KeyEvent`, breaking direct test construction; `encodeRaw` is the testable surface, the Compose-typed overload is the only production caller. (#733)
- **terminal**: Drop unused `MutableSharedFlow` plumbing (`keyEchoSignals`) from `StubTerminalGridSource` and `LoopbackTerminalServer`; `TerminalRenderDevWindow` now drains `source.shellEvents()` so the stub's `PROMPT_STARTED` doesn't accumulate in an uncollected flow. (#733)
- **tests**: 51 tests added across `boss-ipc` + `terminal.render`. `TerminalIpcCompatTest` pins 1.0.0 ↔ 1.1.0 cross-version behaviour and round-trips the new messages. `GridStateTest` covers full redraw, incremental application, revision gap rejection + recovery, attr preservation, style interning, cursor frames, and the duplicate / out-of-order / incremental-before-init cases that originally surfaced the revision-logic bug. `KeyEventEncoderTest` and `ThemePaletteTest` are new. `TerminalGridConnectionTest` stands up the loopback server and exercises the full wire path including the `adopt()` lifecycle. (#733)
