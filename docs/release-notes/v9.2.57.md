# BOSS v9.2.57

**Release Date:** 2026-07-23

Brings Safari-style tab shrinking to the top tab bar — tabs now scale uniformly to fit the available width and only scroll once they hit a favicon-sized floor — with a **Settings → Window** toggle to choose between Shrink to Fit and Fixed Width sizing (#26).

## Highlights

- **Safari-style tab shrinking**: every top tab shrinks uniformly to fit the row width instead of overflowing at a fixed size — 1 tab fills 240dp, ~10 tabs in a 1000dp row settle at ~92dp each, and only at ~30 tabs do they clamp to a 36dp favicon-only floor and the row start scrolling (#26).
- **Tab Sizing setting**: a new **Settings → Window** toggle picks between **Shrink to Fit** (default, the new Safari behavior) and **Fixed Width** (legacy intrinsic sizing) (#26).
- **Smoother, cheaper layout**: per-tab width is measured via `onSizeChanged` on the `LazyRow` rather than a nested `BoxWithConstraints` — avoiding the double-`SubcomposeLayout` that pinned the EDT at 100% CPU past ~10 tabs — and computed in integer pixels so total content is provably ≤ row width and the scrollbar no longer flickers on every tab add (#26).

## Features

- **tabs**: add a **Settings → Window** Tab Sizing toggle — Shrink to Fit (default) vs Fixed Width (#26)

## Improvements

- **tabs**: shrink top tabs uniformly to fit the row (Safari behavior), clamping to a 36dp favicon floor and scrolling only when tabs would drop below it; `BossTabButton` gains a nullable `tabWidth` for exact sizing while `null` preserves legacy intrinsic sizing (#26)
- **tabs**: measure per-tab width via `onSizeChanged` on the `LazyRow` instead of a nested `BoxWithConstraints`, eliminating the double-`SubcomposeLayout` EDT-pinning that hit 100% CPU past ~10 tabs (#26)
- **tabs**: compute per-tab width in integer pixels (floors) rather than Dp (rounds) so total content stays ≤ row width and the scrollbar stops flickering on every tab add (#26)
- **tabs**: make the "+" new-tab button a permanent sibling outside the `LazyRow` so it no longer scrolls off-screen the moment the strip is dragged (#26)
