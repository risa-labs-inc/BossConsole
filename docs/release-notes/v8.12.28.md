# BOSS v8.12.28 Release Notes

Released: December 2025

## Summary

This release improves network reliability by separating API calls from large file downloads, ensuring fast failure detection for API requests while maintaining robust download support.

## Highlights

- Split HttpClient configuration for API calls vs downloads with optimized timeouts (#141)
- API calls now fail fast (30s timeout) instead of waiting unnecessarily long
- Large file downloads maintain 15-minute timeout for slow connections
- Improved network error handling and logging clarity

## Bug Fixes

### Downloads
- **HttpClient Timeout Optimization**: Split HttpClient into separate apiClient and downloadClient with different timeout profiles. API calls use 30s timeout for fast GitHub API interactions, while downloads retain 15min timeout needed for large files (~275MB) on slow connections (Issue #135, #141)
- **Fast Failure Detection**: GitHub API calls now fail quickly (30s) when network issues occur instead of waiting the full 15-minute download timeout (#141)

## Improvements

### Downloads
- **Enhanced Logging**: Updated download timeout logging to clarify configuration and distinguish between API and download operations (#141)
