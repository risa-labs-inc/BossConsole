# BOSS v8.13.6

**Release Date**: November 2025

## Summary

This release introduces comprehensive CLI functionality for BOSS, enabling command-line control of the application with support for Homebrew installation on macOS.

## Highlights

- Complete CLI interface with commands for URL, file, folder, terminal, workspace, and plugin operations (#215)
- Homebrew Cask installation support with automatic CLI setup
- Smart argument detection for natural CLI usage (e.g., `boss google.com`)
- Automatic CLI version management and silent updates

## Features

- **CLI Interface**: Added comprehensive command-line interface using Clikt library (#215)
  - Commands: `url`, `file`, `folder`, `terminal`, `workspace`, `plugin`
  - Smart argument detection automatically detects URLs, files, and folders
  - Command queueing ensures proper execution during app initialization
  - Security validation with path traversal protection and URL validation

- **Homebrew Installation**: Enabled CLI installation via Homebrew Cask (#215)
  - CLI script embedded in app bundle at `BOSS.app/Contents/Resources/boss`
  - Automatic symlink creation at `/opt/homebrew/bin/boss`
  - DMG installations support "Install BOSS CLI" menu option
  - CLI updates automatically when app updates

- **CLI Version Management**: Automatic version tracking and updates (#215)
  - Version headers embedded in CLI scripts during build
  - Silent auto-update on app startup if version mismatch detected
  - Gradle task `generateVersionedCLIScripts` generates versioned scripts

- **Panel Commands**: Added `boss plugin <panelId>` command to open any panel (#215)
  - Opens any of the 14 available panels from command line
  - Supports: bookmarks, terminal, codebase, secret-manager, and more
  - `boss folder` command now automatically opens codebase panel

## Improvements

- **CLI Usability**: Smart detection makes CLI more intuitive (#215)
  - `boss google.com` auto-detects and adds https://
  - `boss file.txt` opens files if they exist
  - `boss .` expands relative paths to absolute paths
  - `boss folder` defaults to home directory when no path provided

- **Cold Start Handling**: Fixed race conditions during app initialization (#215)
  - URLs, files, terminals, and workspaces now load after Last Session
  - 500ms delay ensures tab creation completes before workspace restoration
  - Panel open events use replay buffer to prevent loss during startup

- **CLI Architecture**: Well-structured implementation following best practices (#215)
  - Event bus pattern (URLEventBus, FileEventBus, TerminalEventBus, WorkspaceEventBus)
  - Service coordination (FileHandlerService, WorkspaceHandlerService, TerminalHandlerService)
  - Thread-safe operations using Dispatchers.Main
  - Command queueing with processing state tracking

## Bug Fixes

- **Terminal**: Fixed terminal tab creation timing to wait for session load (#215)
- **Workspace**: Prevented workspace loading from destroying CLI-created tabs (#215)
- **URL Handling**: Fixed URL events being dropped during cold start due to focus state (#215)
- **Panel Lookup**: Fixed panel lookup to compare only identity fields, not display metadata (#215)
- **Logo Function**: Fixed crash with single-character project names like '.' (#215)
- **Path Resolution**: All CLI scripts now expand relative paths to absolute paths (#215)

## Dependencies

- **Added**: Clikt 4.2.1 for CLI argument parsing (#215)
