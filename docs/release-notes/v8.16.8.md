# BOSS v8.16.8 Release Notes

**Release Date:** 2026-01-25

## Summary

This release introduces a comprehensive structured logging framework with sensitive data sanitization, fixes critical browser scrolling issues on macOS, optimizes editor screen space, and enhances password management with discrete fill modes.

## Highlights

- New `BossLogger` framework with automatic sanitization of sensitive data (emails, tokens, credentials)
- Fixed diagonal scroll jitter on macOS trackpad in browser
- Editor gutter width reduced by 35% for more editing space
- Discrete password fill mode for enhanced privacy

## Features

### Logging
- **Add BossLogger framework with sensitive data sanitization** (#540)
  - Structured logging system built on SLF4J with component-scoped loggers
  - Automatic sanitization of sensitive data (emails, tokens, credential IDs, URIs)
  - Category-based logging (AUTH, PASSKEY, BROWSER, TERMINAL, NETWORK, UI, SYSTEM, EDITOR, FILE, WORKSPACE, GENERAL)
  - Configurable log levels via `BOSS_LOG_LEVEL` environment variable or `boss.log.level` system property
  - Replaces unsafe `println()` and `printStackTrace()` calls across codebase

### Secret Manager
- **Add discrete password fill mode for privacy** (#539)
  - New fill mode option that provides additional privacy when filling passwords
  - Helps prevent shoulder surfing and accidental exposure of sensitive credentials
  - User-configurable preference in secret manager settings

## Bug Fixes

### Browser
- **Fix diagonal scroll jitter on macOS trackpad** (#258) (#548)
  - Resolved scrolling instability when using diagonal gestures on macOS trackpad
  - Smooth scrolling experience restored for all scroll directions
  - Improved browser usability on macOS systems

### Editor
- **Reduce gutter width by 35% for more editing space** (#547)
  - Optimized line number gutter width to reclaim horizontal screen space
  - More room for code without sacrificing gutter functionality
  - Better experience on smaller displays and laptop screens

### UI
- **Move panel content when dragging open plugins** (#535) (#538)
  - Fixed layout issue where panel content wasn't properly repositioned during drag operations
  - Panels now smoothly adjust when plugins are opened via drag-and-drop
  - Improved visual feedback during panel reorganization
