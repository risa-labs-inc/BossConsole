# BOSS v9.2.35

**Release Date:** 2026-07-08

Host integration for the Tool Evolver plugin (formerly Tool Sidecar): new sidebar panel ⋮ menu items ("Open Evolver", "Report Issue") dispatched through the RBAC-filtered `McpToolRegistry` with no compile-time coupling, split-open APIs for tabs and URLs, plus fixes for plugin directory/project providers, the directory picker dialog, and a spurious crash dialog on plugin hot-swap (#838).

## Highlights

- **"Open Evolver" and "Report Issue" in the panel ⋮ menu**: Both items appear only while the Tool Evolver plugin is active — gated on its MCP tools being present in the RBAC-filtered `McpToolRegistry` and routed through `McpToolRegistryImpl.invoke`, so the host gains no compile-time coupling to the plugin. "Open Evolver" is further gated on the permission-gated `evolver_evolve` tool (the user holds the evolve permission, or is admin), while "Report Issue" stays available to everyone with the plugin via the ungated `evolver_open`. Failures surface a UI warning and a `StatusMessageManager` message instead of being silently discarded (#838).
- **Split-open APIs for plugins**: `SplitViewOperations.openTabInSplit` and `openUrlInSplit` are mirrored into plugin-api-core and implemented in `SplitViewOperationsImpl`, letting plugins open a tab or URL into an existing/new split — backing the Tool Evolver's "new tab vs split" and issue-link "Open Link" choosers (#838).
- **Plugin picker/provider fixes**: `TrackingPluginContext` and `SandboxedPluginContext` now forward `directoryPickerProvider` and `projectDataProvider` (previously they fell through to the interface default `null`, so "Open Project" and the Evolve "Browse" button silently did nothing), and the directory dialog is owned by the active window with `alwaysOnTop` so it no longer opens behind the app on macOS (#838).
- **No more crash dialog on plugin hot-swap**: Hot-swapping a plugin jar drops the terminal-tab MCP server's ktor writer socket, surfacing `ClosedWriteChannelException: Broken pipe` — the global `CrashHandler` treated it as fatal. A new `isIgnorable()` walks the cause chain and logs+swallows closed-channel / broken-pipe / connection-reset / cancellation exceptions; genuine faults still report normally (#838).

## Features

- **panel**: "Open Evolver" and "Report Issue" items in the sidebar panel ⋮ menu, gated on the Tool Evolver plugin's MCP tools (permission-gated `evolver_evolve` for Open Evolver, ungated `evolver_open` for Report Issue) and dispatched through the RBAC-filtered `McpToolRegistry` with no compile-time coupling (#838)
- **split**: `SplitViewOperations.openTabInSplit` and `openUrlInSplit` mirrored into plugin-api-core and implemented in `SplitViewOperationsImpl`, so plugins can open a tab or URL into an existing/new split (#838)
- **api**: mirror `ConsoleLogsAPI` + `PluginLogMatcher` into plugin-api-core (api 1.0.59) so the Console plugin's per-plugin log filter and the Tool Evolver's shared log attribution resolve from the host binary (#838)

## Bug Fixes

- **plugins**: `TrackingPluginContext` now forwards `directoryPickerProvider` and `projectDataProvider`, which previously fell through to the interface default `null` — the codebase panel's "Open Project" and the Tool Evolver's Evolve "Browse" button silently did nothing (#838)
- **sandbox**: `SandboxedPluginContext` also forwards `directoryPickerProvider`/`projectDataProvider`, so sandboxed plugins' Evolve "Browse" no longer hits a null picker ("Directory picker unavailable") (#838)
- **picker**: own the directory dialog to the active window and set `alwaysOnTop` so it surfaces in front of the app instead of opening behind the Compose window on macOS (#838)
- **crash**: don't show the "BOSS Has Crashed" dialog for benign dropped-socket exceptions on plugin hot-swap — `CrashHandler.isIgnorable()` logs and swallows closed-channel / broken-pipe / connection-reset / cancellation exceptions (#838)
