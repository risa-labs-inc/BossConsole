# BOSS v8.17.5

**Release Date:** 2026-02-21

Achieves full plugin API parity by closing all 42 plugin API gaps across editor, terminal, and platform categories, and fixes installed plugin version tracking for system and bundled plugins.

## Highlights

- **Complete Plugin API Parity**: All 42 plugin API gaps are now closed — editor APIs (minimap, clipboard, search state, undo/redo, code completion, custom gutters, color schemes), terminal APIs (settings, session events, tab management, split panes, reactive state, working directory), and platform APIs (file dialogs, keyboard shortcuts, diagnostics, cache control, task management)
- **BossEditor v1.0.2 & BossTerm v1.0.91**: Plugin parity required upgrading both libraries to expose new APIs for code completion providers, custom gutter decorations, custom color schemes, terminal split panes, reactive tab state flows, and hyperlink deep processing
- **Plugin Version Fix**: Installed plugin version was `null` in `installed.json` for system and bundled plugins; now populated from the JAR manifest with retroactive backfill on config load

## Features

- **plugins**: Implement plugin parity gap analysis — close all 42 plugin API gaps across editor (E5–E15), terminal (T1–T13), and other (O5–O14) categories (#611) (#638)
  - **editor**: Minimap visibility/width, line spacing, ligatures, antialiasing, native editor toggle, search state, undo/redo, code completion providers, custom gutter decorations, custom color schemes
  - **terminal**: Font size/family settings, session events, tab listing/switching, `createTab`, working directory, split panes, reactive `Flow`-based state, shell utilities, hyperlink info
  - **platform**: Clipboard read/write, file open/save dialogs, keyboard shortcut enumeration, diagnostic reporting, plugin cache control, background task management

## Bug Fixes

- **plugins**: Persist installed plugin version from JAR manifest; backfill `null` versions on config load by reading `META-INF/boss-plugin/plugin.json` from installed JARs (#637)

## Dependencies

- `com.risaboss:bosseditor` 1.0.1 → 1.0.2 (#638)
- `com.risaboss:bossterm-compose` 1.0.90 → 1.0.91 (#638)
