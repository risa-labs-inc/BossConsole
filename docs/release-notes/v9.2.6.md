# BOSS v9.2.6

**Release Date:** 2026-06-03

Lets the plugin manager apply an update to a plugin's already-open instances: the host can now count, reset (reload + close all open tabs of a plugin), and restart, so a freshly-updated plugin no longer keeps running its old code in background tabs.

## Highlights

- **Reset/restart a plugin's running instances on update** (#765): new `PluginLoaderDelegate` entries — `getRunningInstanceCount`, `resetPluginInstances`, and `restartApplication` — let the plugin manager push an update into its open instances. Reset enumerates *every* open tab of a plugin across all panels and windows (via `SplitViewStateRegistry` + `TabSandboxRegistry`, mapping `typeId → sandbox.pluginId`), closes them on the EDT before reloading so no tab outlives its plugin's unload, then reloads; restart goes through `ApplicationRestarter.scheduleRestart()`. Pairs with boss-plugin-api 1.0.39 and plugin-manager 1.4.44.
- **Correct per-plugin tab tracking** (#765): instance enumeration now covers all of a plugin's open tabs rather than only the active one (non-active tabs aren't composed, so they were previously never counted), fixing under-counted instances and background tabs left running stale code. `PluginCrashRegistry.unregisterActiveTab(tabId)` keeps its tab-id guard, which also fixes a crash-path bug where an older tab's dispose could wipe a newer tab's mapping.

## Features

- **plugins**: `PluginLoaderDelegate` gains `getRunningInstanceCount`, `resetPluginInstances`, and `restartApplication`, so the plugin manager can apply a plugin update to its open instances — count, reload + close all of the plugin's tabs, or restart the app via `ApplicationRestarter.scheduleRestart()`. (#765)

## Bug Fixes

- **plugins**: instance tracking now enumerates every open tab of a plugin (across all panels and windows), not just the active one, so `getRunningInstanceCount` no longer under-counts and `resetPluginInstances` no longer leaves background tabs running the old code. (#765)
- **plugins**: `PluginCrashRegistry.unregisterActiveTab(tabId)` takes the tab id and guards on it, preventing an older tab's dispose from wiping a newer tab's active-tab mapping. (#765)

## Dependencies

- boss-plugin-api 1.0.39
- plugin-manager 1.4.44
