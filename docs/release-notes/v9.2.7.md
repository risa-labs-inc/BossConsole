# BOSS v9.2.7

**Release Date:** 2026-06-03

Fixes "Restart BOSS" silently quitting without relaunching on packaged (jpackage) installs — the path the plugin manager's post-update restart prompt invokes — so the app now reliably comes back up after a restart.

## Highlights

- **"Restart BOSS" now relaunches packaged installs** (#767): `restartApplication()` previously relaunched with `java -jar <single-jar>`, which can't boot a packaged app (wrong classpath, missing native libs like JxBrowser/Skia) — so the old instance exited and nothing started. It now relaunches the native install instead: on macOS it detects the running `.app` bundle and re-opens it via LaunchServices; on Windows/Linux it re-execs the jpackage native launcher with its original args. The relaunch waits for the old PID to exit before starting so a fresh instance is spawned rather than the dying one re-activated. Dev (JAR/Gradle) fallbacks are unchanged.

## Bug Fixes

- **runner**: "Restart BOSS" no longer quits without coming back on packaged installs. `ApplicationRestarter.scheduleRestart()` relaunches the native install (macOS `.app` via LaunchServices; Windows/Linux jpackage launcher re-exec) instead of `java -jar`, and waits for the old PID to exit first. Windows uses a PowerShell `Get-Process` wait loop to avoid racing `SingleInstanceManager` (which would otherwise see the old PID still alive and `exitProcess(0)`), and `detectMacAppBundle` validates each candidate with `File.exists()` so a stray `.app` substring on the launcher path or `java.library.path` can't be handed to `open`. (#767)
