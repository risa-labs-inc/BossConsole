# BOSS v9.2.8

**Release Date:** 2026-06-03

Fixes the auto-update progress bar jumping from 0% straight to 100% by switching the updater to a streaming download, so the download banner now reflects bytes actually arriving off the socket.

## Highlights

- **Smooth auto-update progress** (#768): The updater used Ktor's buffering `get()`, which read the entire response body into memory before the progress read-loop ran — so the bar sat at 0% and then jumped to 100%. `downloadUpdate()` now streams via `prepareGet().execute { }` through a shared `streamToFile()` helper that reads `bodyAsChannel()` inside the execute lambda (the same pattern already used in `RemotePluginRepository`), so progress tracks real download throughput.

## Bug Fixes

- **updater**: Auto-update download progress no longer sits at 0% and then snaps to 100%. `downloadUpdate()` streams the response with `prepareGet().execute { }` + `bodyAsChannel()` instead of buffering the whole body via `get()`, and the buggy unknown-size sawtooth fallback (`0.1f + bytes/1MB % 0.8f`, which reset every ~0.8MB) is replaced with a monotonic asymptotic curve. `BossLogger` logging and typed-exception handling are preserved; no UI or state changes. (#768)
