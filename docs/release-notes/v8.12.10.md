# BOSS v8.12.10

**Release Date**: October 2024

## Summary

This release focuses on plugin architecture cleanup and database schema consolidation, removing 14 unimplemented placeholder panels and replacing 42 incremental migrations with a single comprehensive baseline.

## Highlights

- Removed 14 placeholder "To be implemented" panels, reducing codebase by ~1,176 lines
- Reorganized plugin panel layout with functional grouping (RPA tools and admin panels)
- Consolidated 42 migration files into single comprehensive schema matching production
- Fixed critical signup bug in handle_new_user() function
- Replaced hardcoded encryption key with Supabase Vault for proper security

## Improvements

### Plugin Architecture
- **Removed 14 unimplemented placeholder panels** (#89)
  - Deleted left-top panels: Lighthouse, Value/North Star, SystemOfRecord
  - Deleted left-bottom panels: Lanager, Mastery, TaskResolver
  - Deleted bottom panels: BugReport, Activity, Errors, Git
  - Deleted right-top panels: Docker, Database, Agent
  - Deleted right-bottom panels: EhrExplorer
  - Cleaner plugin registry with only functional plugins (10 panels + 3 tab types)

- **Reorganized panel layout for better grouping** (#89)
  - Moved all panels to unified right area
  - Right-top area: RPA & Browser panels (Fluck, LLM RPA, RPA Recorder, RPA Engine)
  - Right-bottom area: Admin & Security panels (Roles, Create Roles, Secret Manager)
  - Clear functional separation and improved navigation

- **Fixed duplicate panel ID conflicts** (#89)
  - Resolved AdminRoleManagement and RpaEngine both using ID 20
  - Resolved RoleCreation and SecretManager both using ID 21
  - Each panel now has unique ID for predictable ordering

### Database Schema
- **Consolidated migrations from production database** (#88)
  - Replaced 42 incremental migration files with single comprehensive schema dump
  - Reduced migration file size by 32.4% (2,796 → 1,888 lines)
  - Added 44 section/subsection comments for navigation
  - Established clean baseline matching production exactly

- **Added seed data for default system roles** (#88)
  - Automatically populates 'user' and 'admin' roles after database reset
  - Ensures roles exist before any user signs up
  - Makes database resets safer and more reproducible

- **Removed obsolete shell scripts** (#88)
  - Deleted set-jwt-secret.sh (one-time setup, already configured)
  - Deleted test-deployed-jwt.sh (outdated test utility)
  - Cleaner project structure

## Bug Fixes

### Database
- **Fixed critical signup bug in handle_new_user() function** (#88)
  - Bug referenced non-existent 'role' column instead of 'role_id'
  - Now correctly uses UUID lookup from roles table
  - This bug would have caused ALL new user signups to fail

### Security
- **Replaced hardcoded encryption key with Supabase Vault** (#88)
  - Master encryption key was hardcoded and visible in git repository
  - Updated get_encryption_key() to retrieve key from Supabase Vault
  - Key now stored encrypted at rest using pgsodium
  - Added VAULT_SETUP.md with complete setup and key rotation procedures
  - Industry-standard security practices for handling sensitive user data
