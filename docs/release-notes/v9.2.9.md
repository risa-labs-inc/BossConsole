# BOSS v9.2.9

**Release Date:** 2026-06-04

Brings sidebar plugins up to parity with main tabs: a header right-click menu, "Open as Tab" promotion, a draggable header, and per-plugin update indicators — plus workspace-restore hardening so unknown/legacy tab types no longer crash restore.

## Highlights

- **Open a sidebar plugin as a tab** (#772): "Open as Tab" *moves* a sidebar plugin into a focused main tab (single cached instance, state preserved). Clicking the sidebar icon while it's hosted focuses that tab; closing the last hosting tab reopens it in its sidebar location.
- **Draggable plugin header** (#772): Drag a sidebar plugin's header to reorder within the sidebar, or drop it onto the central area to promote it to a tab — reusing the tab-drag drop zones (center = add as tab, edge = vertical/horizontal split) with a matching drop highlight.
- **Per-plugin update indicator** (#772): Each sidebar header shows an "Update" badge when a host-compatible newer version exists, plus a "Check for Updates" action in the kebab and right-click menus.

## Features

- **panel**: Right-clicking a sidebar plugin's header now opens the same menu as the "…" kebab (Restart Panel / Reload Plugin / Open as Tab / Minimize), built from one shared action list. (#772)
- **panel**: "Open as Tab" moves a sidebar plugin into a focused main tab as a single cached instance with state preserved; the sidebar icon focuses the hosted tab, and closing the last hosting tab restores it to the sidebar (count-based hosted tracking). New `PanelHostTab` host-internal tab type and `PromoteSidebarToTab` deferred promote/focus effects. (#772)
- **panel**: The plugin header is now draggable — reorder/move within the sidebar, or drop onto the central area to promote to a tab with split-pane detection (panel center = add as tab, edge = create vertical/horizontal split). (#772)
- **panel**: Sidebar headers surface plugin updates — an "Update" badge when a host-compatible newer version exists (click to update) and a "Check for Updates" action that prompts on a compatible update, else reports "up to date" / "needs a newer BOSS". Wired via `PluginUpdateRegistry`, `LocalPanelPluginIdResolver`, and a `PluginUpdateBridge` expect/actual delegating to the existing `PluginUpdateManager` + `DynamicPluginManager`; the startup refresh now feeds installed plugins to the updater. (#772)

## Bug Fixes

- **panel**: Workspace restore no longer crashes with "IllegalArgumentException: Unknown tab type" — `WorkspaceExtractor` stops persisting transient panel-host tabs and `WorkspaceApplier` skips unknown/legacy tab types instead of throwing. The split path also resolves the first tab before calling `splitPanel`, so an unresolved legacy tab no longer leaves an empty "ghost" split panel on recovery. (#772)
- **panel**: Panel-host tabs can no longer double-compose the shared cached component. The host tab is strictly single-instance; dragging an already-hosted plugin's icon to the center now focuses the existing tab, and `splitPanel` *moves* a panel-host tab (rather than copying it) so Split Right/Down or a same-panel edge drop won't render the same component in two tabs. (#772)
- **panel**: Hosted-as-tab tracking is now a true reference count (`Map<PanelId, Int>`), and split-move adds the new hosting tab before removing the original (count 1→2→1, never 0) so a move sequence can't transiently unhide the sidebar panel. `TabUpdateRegistry.unregisterTab` is ownership-checked so the source's close can't wipe the destination's mapping for the same fixed tab id. (#772)

## Improvements

- **panel**: Renamed the sidebar header action "Reset" → "Restart Panel" with a distinct `RestartAlt` icon, making its scope (this panel, fresh state, same code) explicit next to "Reload Plugin". (#772)
- **panel**: `PluginUpdateRegistry` now uses atomic `MutableStateFlow.update {}` for put/clear and an atomic merge (`putAll`) for `setAll`, so a startup `refreshAll` can't clobber a concurrent per-window `checkOne`. Each check reads its own `UpdateCheckResult` instead of the shared `availableUpdates` flow, and a failed single check reports an error instead of clearing a known update. (#772)
- **security**: `performUpdate` sanitizes the download filename and verifies the resolved path stays inside the plugin directory, since a version string from the remote store manifest can carry path separators that `SemanticVersion.parse` does not reject. (#772)
