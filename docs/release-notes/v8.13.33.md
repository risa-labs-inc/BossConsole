# BOSS v8.13.33

**Release Date**: December 2025

## Summary

This release focuses on critical browser stability improvements, including a new Reset Browser feature to recover from persistent JxBrowser issues, thread safety fixes, and enhanced update installation experience with streamlined Linux authentication prompts.

## Highlights

- New Reset Browser feature in Help menu to clear browser profile and recover from crashes
- Critical thread synchronization fix preventing race conditions in engine initialization
- Help menu with Check for Updates option for easier version management
- Streamlined Linux update process with single authentication prompt

## Features

### Browser
- **Reset Browser option to fix persistent JxBrowser issues** (#348, c97ba460)
  - Added "Reset Browser..." menu item in Help menu with confirmation dialog
  - Clears cache, cookies, and sessions without restarting application
  - Reset Browser button shown in error view when max retries exceeded
  - Cross-platform support with expect/actual declarations
  - Related to #340

### Updates
- **Help menu with Check for Updates option** (#344, ba1e0fcf)
  - New Help menu added to MenuBar (after Window menu)
  - "Check for Updates..." menu item triggers UpdateManager
  - Consistent with BossTerm's Help menu implementation

## Bug Fixes

### Browser
- **Thread synchronization in FluckEngine.engine getter** (#348, 0bb738ed)
  - Fixed critical race condition where multiple threads could simultaneously detect closed engine
  - Prevented resource leaks from duplicate engine initialization
  - Wrapped getter logic in synchronized block for atomic check-and-update operations
- **Thread safety in resetBrowserProfile** (#348, 5acda976)
  - Made resetBrowserProfile a suspend function with Dispatchers.IO
  - Prevents UI freeze (50-500ms+) when called from dialog
  - Added ResetResult data class for detailed step tracking
  - Comprehensive error handling with no silent failures
  - Reference: docs/THREADING.md:24
- **Graceful handling of closed JxBrowser engine** (#348, da80e4ac)
  - Check if cached engine is closed before returning
  - Auto-reinitialize engine if closed during app restart
  - Pre-flight checks in getBrowserState() before creating browser
  - Improved error diagnostics for closed object exceptions
  - Prevents retry loops when engine is unavailable

### Updates
- **Batch Linux update sudo prompts into single authentication** (#344, 9339d71a)
  - Merged all privileged operations into single pkexec/sudo call
  - Reduces password prompts from 2-4 down to 1
  - Applies to both DEB and RPM updates
  - Added platform-file type validation before installation
- **Improve update installer robustness** (269e4972)
  - Routes installer by file extension instead of platform detection
  - Handles cases where .deb/.rpm unavailable but .jar is present
  - Added JAR fallback in asset selection for Linux

## Improvements

### General
- **Update app name to Business Operating System + Simulation** (76309c3d)
  - Window title: "BOSS - Business Operating System + Simulation"
  - Updated CLI help text, Linux .desktop file, Windows registry

## Dependencies

- **BossTerm updated to 1.0.54** (269e4972)
  - Enhanced terminal capabilities and stability
