# BOSS v8.12.8

**Release Date**: October 2024

## Summary

This release introduces a complete dynamic Role-Based Access Control (RBAC) system with runtime role and permission management, eliminates the ENUM-based authorization architecture in favor of a flexible table-based approach, and includes critical fixes for passkey authentication refresh token handling.

## Highlights

- Dynamic role and permission creation system allowing admins to manage access control without code changes
- Complete RBAC migration from PostgreSQL ENUMs to table-based schema with full CRUD support
- Fixed critical passkey refresh token bug that logged users out after ~1 hour
- New Role Creation admin panel with comprehensive UI for role/permission lifecycle management

## Features

### Dynamic Role & Permission Management System (#86)

- **Runtime role creation** - Admins can create custom roles (e.g., `ai_trainer`, `code_reviewer`) without deploying code
- **Runtime permission creation** - Define new permissions dynamically following `domain.action` pattern
- **Role-permission mapping** - Assign and revoke permissions from roles via UI
- **System role protection** - Built-in `user` and `admin` roles cannot be deleted
- **Validation and security** - Regex validation for role/permission names, admin-only RPC functions

### Database Architecture Overhaul

- **Table-based RBAC schema** - Replaced ENUMs with `roles` and `permissions` tables using UUID foreign keys
- **17 comprehensive RPC functions** - Complete API for role/permission operations with CASCADE delete support
- **Metadata support** - Roles and permissions now include descriptions, `is_system` flags, and timestamps
- **Auth hook integration** - Custom access token hook injects RBAC claims into JWTs from table data

### Role Creation UI Panel (#86)

- **Compact admin panel** - Vertical stacked layout optimized for narrow sidebar
- **Role selection dropdown** - View all roles (system + custom) with real-time permission display
- **Create role dialog** - Name validation, description field, instant feedback
- **Create permission dialog** - `domain.action` format validation, error handling for duplicates
- **Assign/revoke permissions** - Checkbox-based permission management per role
- **Delete operations** - Remove custom roles/permissions with confirmation dialogs
- **System role badges** - Visual indicators preventing accidental deletion of system roles

### Updated User Role Management

- **Dynamic role assignment** - Assign any role (enum or custom) to users from dropdown
- **String-based role display** - User role badges now show all assigned roles including custom ones
- **Backward compatibility** - Legacy `AppRole` enum removed, simplified to String-based API

## Bug Fixes

### Passkey Authentication (#78)

- **Fixed refresh token expiration bug** (#75)
  - Users no longer logged out after ~1 hour
  - Replaced manual JWT generation with Supabase Admin API approach
  - Refresh tokens now stored as unique strings in `auth.sessions` table
  - Enables automatic token rotation and single-use tokens

- **Fixed passkey JWT generation** (#86)
  - Updated `getUserRoles()` in jwt.ts to query table-based schema (not ENUMs)
  - Admin role now correctly included in passkey authentication JWTs

- **Removed misleading error handling** (#78)
  - Token generation failures now properly propagate with `success: false`
  - Eliminated silent authentication failures

### RBAC System

- **Fixed role assignment for custom roles** (#86)
  - Custom roles (e.g., `ai_trainer`) now display in User Role Management UI
  - Changed `UserWithRoles.roles` from `List<AppRole>` to `List<String>`
  - Updated `UserService` to preserve dynamic role names

- **Fixed JSON deserialization errors** (#86)
  - Added missing `permission`, `description`, `permission_id`, `role`, `role_id` fields to `RpcResponse`
  - Resolved "unknown key" errors during permission/role creation

- **Fixed null safety issues** (#86)
  - Replaced dangerous `!!` operators with `getOrThrow()` and `.onSuccess/.onFailure` patterns
  - Prevents potential null pointer crashes

### Database Functions

- **Force-updated RPC functions** (#86)
  - `get_all_roles()`, `get_all_permissions()` now return proper `{success, data: [...]}` format
  - Fixed serialization errors where Kotlin expected `data` but received `roles`
  - All functions query tables instead of ENUMs

## Breaking Changes

### RBAC API Migration

**Old ENUM-based API:**
```kotlin
AuthService.currentUserHasRole(AppRole.ADMIN)
AuthService.assignRole(userId, AppRole.ADMIN)
```

**New String-based API:**
```kotlin
AuthService.currentUserHasRole("admin")
AuthService.assignRoleByName(userId, "admin")
```

**Impact:**
- All role/permission checks now use String literals
- Custom roles work seamlessly without code changes
- Plugins can create their own roles dynamically
