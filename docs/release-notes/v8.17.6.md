# BOSS v8.17.6

**Release Date:** 2026-03-02

Maintenance release fixing a macOS fullscreen animation glitch when closing the window, removing the deprecated plugin-api module, and updating CI/CD dependencies.

## Highlights

- **macOS Fullscreen Close Fix**: Closing the app from a non-fullscreen state no longer triggers a spurious fullscreen animation; the fix detects actual native fullscreen state before calling `requestToggleFullScreen` (#646)
- **Deprecated Plugin API Removed**: The legacy `plugin-api` module has been deleted — its functionality is fully superseded by `plugins/plugin-api-core`, which adds logging, caching, background tasks, keyboard shortcuts, and typed tab APIs (#645)
- **BossTerm 1.0.92**: Terminal library patch update (#641)
- **CI/CD Action Updates**: `actions/upload-artifact` v6 → v7, `actions/download-artifact` v4 → v8, and `digicert/ssm-code-signing` v1.1.1 → v1.2.1 (#644, #642, #643)

## Bug Fixes

- **window**: Prevent macOS window from entering fullscreen animation on close — compare window bounds to screen bounds before calling `requestToggleFullScreen`, only toggling when actually in native fullscreen (#646)

## Improvements

- **plugins**: Remove deprecated `plugin-api` module; all capabilities are available in `plugins/plugin-api-core` (#645)

## Dependencies

- `com.risaboss:bossterm-compose` 1.0.91 → 1.0.92 (#641)
- `actions/upload-artifact` 6 → 7 (#644)
- `actions/download-artifact` 4 → 8 (#642)
- `digicert/ssm-code-signing` 1.1.1 → 1.2.1 (#643)
