# BOSS v9.2.19

**Release Date:** 2026-06-30

Makes Supabase the primary desktop self-update channel with Realtime push notifications and an automatic GitHub Releases backup, replacing the old polling-only flow. Also documents the "Operator's Console" design system in the README with a durable, committed visual styleguide.

## Highlights

- **Supabase Realtime update source with GitHub backup**: An `UpdateSource` seam below `DesktopUpdateService` adds `SupabaseUpdateSource` (PostgREST over Ktor) as the primary channel and `GitHubUpdateSource` as an automatic fallback (`FallbackUpdateSource`: Supabase → GitHub). The existing `GitHubRelease` model is reused so all version-selection and asset-matching logic is unchanged (#810).
- **Push-based updates replace polling**: `AppUpdateRealtimeService` subscribes to `app_releases` `postgres_changes` and runs an immediate check on each new release and on (re)connect catch-up; the periodic poll loop is demoted to a startup check plus a disconnect-only safety net (#810).
- **Packaged config from CI secrets**: `SUPABASE_ANON_KEY` / `SUPABASE_URL` / `SUPABASE_FUNCTION_URL` are now baked into the packaged app launcher as `-D` system properties at package time, so a rotated key takes effect on rebuild without a code change (advances issue #33) (#810).
- **Design System documentation**: The README gains a Design System section linking the canonical spec (`docs/DESIGN_SYSTEM.md`) and a self-contained, committed visual styleguide (`docs/design-system.html`) (#809).

## Features

- **updater**: Supabase update source with Realtime push and GitHub backup — new `app_releases` table (catalog + per-platform asset URLs/sha256, app discriminator) with public-read RLS, added to the `supabase_realtime` publication; configurable via `BOSS_UPDATE_BUCKET` / `BOSS_UPDATE_APP_ID` / `BOSS_UPDATE_PRIMARY_SOURCE` (#810)
- **updater**: Optional SHA-256 verification of downloads when the source supplies a hash (#810)
- **updater**: Bake Supabase config into the packaged app from CI secrets, so rotated keys apply on rebuild without code changes (advances issue #33) (#810)

## Bug Fixes

- **updater**: `downloadUpdate` falls back to the GitHub asset when the Supabase Storage download fails (closing the metadata-only fallback gap) (#810)
- **updater**: `UpdateManager` single-flight `checkMutex` coalesces the startup check stampede (#810)
- **updater**: URL-encode PostgREST app/version params (#810)
- **updater**: Realtime loop unsubscribes and backs off before reconnect, avoiding busy-spin and channel leaks (#810)

## Improvements

- **supabase**: Provision a public `app-releases` Storage bucket via an idempotent migration for app binaries (#810)
- **ci**: `sync-release.yml` uploads binaries to Supabase Storage and upserts the `app_releases` row (firing Realtime); a no-op until `SUPABASE_SERVICE_ROLE_KEY` is configured (#810)
- **docs**: Ship the styleguide as committed HTML (`docs/design-system.html`) for a durable link instead of a session-scoped artifact URL (#809)
