# BOSS v9.2.24

**Release Date:** 2026-07-03

The plugin-manager rebrand lands in the host: the setup wizard, menu bar, and Help actions now speak "Toolbox" and "tools" instead of "Plugin Manager" and "plugins." The self-update dialog also renders its "What's new" release notes as real Compose markdown — headings, tables, lists, code blocks, and inline styles — and stops resizing as you scroll through them.

## Highlights

- **Toolbox rebrand in the host UI**: The "Plugin Setup Wizard…" menu item becomes "Toolbox Setup Wizard…", the menu-bar "Plugin" panel list becomes "Toolbox" (with a "(No tools available)" fallback and a "Reload All Tools" Help action), and the wizard's copy switches from plugin(s) to tool(s) with a toolbox hero glyph — internal identifiers and log messages are unchanged (#819).
- **Update dialog renders release notes as markdown**: The "What's new" section now renders natively in Compose — headings, bullet/numbered lists, GFM tables, fenced code blocks, thematic breaks, and inline bold/italic/code/strikethrough/links (clickable) — instead of showing raw markdown as plain text, with a best-effort fallback to the previous plain-text rendering on any parse failure (#820).
- **Stable update dialog while scrolling**: Variable-height markdown blocks no longer make the update dialog grow and shrink as you scroll; a fully composed `Column` + `verticalScroll` behind clamped intrinsics with pinned baselines fixes the height at `min(content, 220dp)` regardless of scroll position (#820).

## Improvements

- **tabs**: Rebrand plugin-manager references to "Toolbox" across the setup wizard — welcome hero copy (now using the `HomeRepairService` glyph), step titles, category descriptions, install/complete summaries, and failure strings all move from plugin(s) to tool(s) (#819)
- **panel**: Menu bar "Plugin" panel-reveal menu is renamed "Toolbox" with a "(No tools available)" empty fallback, and the Help-menu action becomes "Reload All Tools" (#819)

## Bug Fixes

- **updater**: Render the update dialog's "What's new" release notes as Compose markdown (headings, lists, GFM tables, fenced code, thematic breaks, inline styles with clickable links) instead of raw plain text, falling back to the prior plain-text rendering on empty or failed parsing (#820)
- **updater**: Stop the update dialog resizing while scrolling release notes — replace the wrap-content `LazyColumn` with a fully composed `Column` + `verticalScroll` behind a clamped intrinsic-max-height modifier and pinned baselines, fixing dialog height at `min(content, 220dp)` (#820)
