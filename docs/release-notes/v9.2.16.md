# BOSS v9.2.16

**Release Date:** 2026-06-27

Lets plugins introduce their own RBAC permissions: a plugin declares `definedPermissions` in its manifest, the catalog auto-registers them (ungranted) at publish, and permission-hidden plugins now surface to non-admins as an actionable "ask an admin" banner instead of silently vanishing.

## Highlights

- **Plugins can declare new permissions**: A plugin manifest can list `definedPermissions: [{name, description}]` — the new permissions it introduces — and the plugin-store edge function auto-registers each one (non-system, **ungranted**) at publish via the `register_plugin_permission()` SECURITY DEFINER RPC, closing the loop where a plugin could require a permission that didn't exist in the catalog (#797).
- **Inaccessible plugins are visible to non-admins**: `PluginLoaderDelegate.getInaccessiblePlugins()` exposes installed-but-permission-hidden plugins (id, display name, missing permissions) so UIs like the Plugin Manager can show a "requires permission X — ask an admin" banner instead of silently omitting them (#799).
- **Auto-register on the multi-step publish path too**: The local-jar `/publish` route now also reads `requiredPermissions`/`definedPermissions`, rejects dangling required permissions, and registers defined ones — bringing `main` in line with the deployed edge function (#798).

## Features

- **plugin**: Declare plugin-introduced permissions in the manifest (`definedPermissions`) and auto-register them at publish. New `register_plugin_permission()` RPC (migration `20260629000000`) inserts a non-system, ungranted permission with namespacing (`domain.action`) + reserved-domain blocking (`role`/`user`/`api_key`/`rpa`/`secret`/`plugins`), never shadows a system permission, and never grants; a `plugin_permissions` provenance table (readable via `role.read`) records which plugin defined which permission. The publish path rejects a dangling `requiredPermissions` entry not in the catalog. pgTAP coverage runs in CI (#797)
- **plugin**: Expose inaccessible (permission-hidden) plugins via `PluginLoaderDelegate.getInaccessiblePlugins(): List<InaccessiblePluginInfo>` (default empty for back-compat), sourced from `DynamicPluginManager`'s permission-hidden plugins using the existing `missingPermissions(manifest)` helper. boss-plugin-api compile mirror bumped to 1.0.47 (#799)

## Improvements

- **plugin**: Register defined permissions on the multi-step (local-jar) `/publish` route as well — accept `requiredPermissions`/`definedPermissions` in the publish request schema, reject dangling required perms, and generalize the helper param to a `Pick` so both the jar manifest and the request body satisfy it (#798)
- **plugin**: Log the precise *missing* permission subset plus an actionable hint ("Ask an admin to grant: …") when a plugin is hidden from a non-admin, at both hide points (#797)
