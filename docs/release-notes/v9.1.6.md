# BOSS v9.1.6

**Release Date:** 2026-05-12

Maintenance release adding a `workflow_run` fallback so the public-repo release sync fires automatically after Release Build completes (v9.1.4 and v9.1.5 both needed manual `workflow_dispatch`), plus routine dependency bumps across Ktor, Supabase, GraalVM Native Build Tools, kotlinx-datetime, and BossTerm.

## Highlights

- **Auto-sync release pipeline**: `sync-release.yml` now triggers on `workflow_run.completed` from the Release Build workflow as a fallback, with discovery via the GitHub API filtered by `created_at >= run_started_at` so build-only verification runs (`create_release=false`) don't push stale release data to the public README.
- **Global serialization for release sync**: Concurrency group set to a static `sync-release-global` key — the original tag-vs-SHA scheme never collapsed across triggers, so a double-fire could race on the public-repo README push.
- **GraalVM Native Build Tools 1.1.0**: Major version jump from 0.11.5.
- **Supabase 3.6.0**: BOM and all submodules (postgrest, auth, realtime, storage, functions) move from 3.4.1.

## Improvements

- **workflow**: Auto-trigger `sync-release` on Release Build completion via `workflow_run`, discovering the release by querying the GitHub API and gating on `created_at >= workflow_run.run_started_at` so a build-only run with `create_release=false` doesn't republish the prior release. Includes tag-format validation on the workflow_run path, epoch-numeric timestamp compare via `date -d … +%s`, surfacing of `gh api` errors (no silent `2>/dev/null`), a defensive empty-check on `WORKFLOW_RUN_STARTED_AT`, `per_page=10` on the listing call, and a documented migration path (tag-resolution pre-job + `workflow_call`) if a second release pipeline ever appears. (#708)
- **workflow**: Add static `sync-release-global` concurrency group so the `release: published` and `workflow_run` triggers can never race on the public-repo README git push. Earlier iterations keyed on tag-or-SHA but the two triggers never agreed on a group. (#708)

## Dependencies

- **deps**: Bump `org.jetbrains.kotlinx:kotlinx-datetime` from `0.7.1-0.6.x-compat` to `0.8.0-0.6.x-compat`. (#700)
- **deps**: Bump Ktor from `3.4.1` to `3.4.3` (server-core-jvm, server-netty-jvm, server-test-host, client-core, client-cio, client-content-negotiation, serialization-kotlinx-json, and the `io.ktor.plugin` Gradle plugin). (#702)
- **deps**: Bump `org.graalvm.buildtools.native` from `0.11.5` to `1.1.0`. (#703)
- **deps**: Bump Supabase from `3.4.1` to `3.6.0` (bom, postgrest-kt, auth-kt, realtime-kt, storage-kt, functions-kt). (#701)
- **deps**: Bump `com.risaboss:bossterm-compose` from `1.0.92` to `1.0.93`. (#704)
