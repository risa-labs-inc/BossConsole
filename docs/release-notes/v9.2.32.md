# BOSS v9.2.32

**Release Date:** 2026-07-05

macOS launch fix: removes the still-ungranted passkey managed-capability entitlement that broke every macOS launch once the v9.2.31 XML-comment fix (#835) stopped masking it.

## Highlights

- **macOS app launches again**: `com.apple.developer.web-browser.public-key-credential` is a "managed capability" — macOS refuses to open a Developer ID app that declares it without a matching `embedded.provisionprofile`. Apple's approval for this capability (entitlement request `YWD6D3446V`) is still pending, so no profile exists and every macOS launch failed with a generic "The application can't be opened" error. The key is removed from `BOSS.entitlements`; the runtime `webBrowserPasskeyEntitlementProvisioned` guard already treats the platform-passkey feature as unavailable until the profile is embedded, so this changes no runtime behavior (#836).

## Bug Fixes

- **macos**: Remove the ungranted `com.apple.developer.web-browser.public-key-credential` managed-capability entitlement from `BOSS.entitlements` — it required an embedded provisioning profile that does not yet exist and blocked every macOS launch of the signed app (surfaced only after #835 fixed the XML-comment error that had been failing `codesign` earlier). The `BossWebAuthn.swift` provisioning guard keeps the platform-passkey feature a strict no-op until Apple provisions the App ID, so removing the key is runtime-neutral (#836)
