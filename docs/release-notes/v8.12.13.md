# BOSS v8.12.13

**Release Date**: October 2024

## Summary

This release improves passkey authentication UX with better credential selection, proper error isolation between authentication flows, and infrastructure cleanup.

## Highlights

- Enhanced passkey selection with clear credential identifiers
- Fixed error message cross-contamination between magic link and passkey flows
- Added authentication cancellation support
- Removed self-hosted Supabase configuration

## Bug Fixes

### Authentication
- **Improved passkey selection UX and error message isolation** (#100, Issue #72)
  - Added dedicated PasskeySelectionScreen for multi-passkey scenarios with unique credential IDs (first 4 + last 4 chars)
  - Removed confusing transport badges and misleading timestamps from passkey display
  - Fixed bug where "Magic link sent!" success message was shown as error
  - Properly isolated passkey errors from magic link errors to prevent cross-contamination
  - Added `cancelAuthentication()` method to enable "Back to Sign In" during authentication
  - Filter credentials by credentialId for specific passkey selection
  - Set available passkeys from `checkUserExists()` to avoid authenticated API calls during login

### Infrastructure
- **Removed self-hosted Supabase configuration** (#99)
  - Cleaned up unused self-hosted Supabase infrastructure code

## Improvements

### Authentication
- Improved state management for passkey authentication with proper coroutine job cancellation
- Enhanced authentication flow with StateFlow-based credential management
- Added `clearError()` method to CoreLoginViewModel for better error handling
