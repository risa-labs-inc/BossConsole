# BOSS v8.14.0

**Release Date**: December 2025

## Summary

This release introduces runner terminal integration with the sidebar panel, providing unified run execution through BossTerm with play/stop controls and improved state management.

## Highlights

- **Runner sidebar terminal integration** - Execute run configurations directly in sidebar terminal tabs (#364, #347)
- **Play/stop controls** - Runner dropdown with play/stop buttons for active configuration management
- **Unified execution service** - RunnerTerminalService replaces polling with callback-based session listeners
- **BossTerm 1.0.61 integration** - Stable tab ID API for improved terminal state tracking

## Features

### Runner
- **Runner terminal integration with sidebar panel** (#347, #364)
  - Execute run configurations in sidebar terminal tabs
  - RunnerTerminalService for unified run execution
  - Replace polling with callback-based session listener
  - BossTerm 1.0.61 stable tab ID API integration
  - Proper cleanup of terminal mappings to prevent memory leaks

- **Enhanced runner controls** (#347)
  - Play/stop buttons in runner dropdown
  - Rerun button added to dropdown menu
  - Select configuration on action for better UX
  - Stop button closes terminal tab instead of sending Ctrl+C
  - Re-run sends command to correct tab in sidebar mode

- **Improved configuration management** (#347)
  - Include project name in runner configuration display name
  - Preserve project name when disambiguating duplicate config names
  - Better runner selection and project panel behavior
  - Enhanced run config name generation with debug logging

## Bug Fixes

### Runner
- **Memory leak prevention** (#347)
  - Clean up all mappings in markTerminalStopped
  - Proper terminal state cleanup on stop

- **Threading and reliability** (#347)
  - Address critical threading, security, and reliability issues
  - Fix state consistency problems
  - Remove duplicate code

- **Terminal tab management**
  - Re-run sends command to correct tab in sidebar mode
  - Stop button properly closes terminal tab

## Improvements

- **Code quality**
  - Refactored polling to callback-based session listeners
  - Addressed memory and duplicate code issues
  - Enhanced state consistency across runner components
