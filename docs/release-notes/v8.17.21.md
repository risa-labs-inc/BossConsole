# BOSS v8.17.21

**Release Date:** 2026-03-09

Windows ARM64 build fix for the microkernel architecture introduced in v8.17.20: microkernel modules are excluded on Windows ARM64 (where `protoc` has no binary) and `KernelBootstrap` is loaded via reflection, keeping MONOLITH mode fully functional on that platform.

## Highlights

- **Windows ARM64 compatibility**: All microkernel (`boss-*`) modules are conditionally excluded on Windows ARM64 via `settings.gradle.kts`, removing the dependency on `protoc`/`protoc-gen-grpc-java` which do not publish ARM64 binaries
- **Reflection-based boot**: `KernelBootstrap` is now loaded via reflection in `main.kt`, allowing the app to compile without `boss-ipc` classes on unsupported platforms while MONOLITH mode runs unchanged

## Bug Fixes

- **build(windows-arm64)**: Exclude all microkernel modules (`boss-ipc`, `boss-process-manager`, `boss-orchestrator`, etc.) on Windows ARM64 via conditional `settings.gradle.kts` includes; remove 574 pre-generated source files (194K lines) that were committed as a previous workaround
- **build(windows-arm64)**: Add conditional `composeApp` source exclusions for `kernel/` and `remote/` packages on Windows ARM64 and load `KernelBootstrap` via reflection to avoid compile-time dependency on `boss-ipc` classes
