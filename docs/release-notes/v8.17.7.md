# BOSS v8.17.7

**Release Date:** 2026-03-05

Major stability release overhauling plugin crash isolation and recovery with a new PluginCrashRegistry, centralizing app data via BossDirectories, and migrating BossEditor from a local module to a published Maven artifact.

## Highlights

- **Plugin Crash Sandboxing**: A new `PluginCrashRegistry` intercepts Compose composition crashes, shows a styled fallback UI above the corrupted subtree, and closes the crashed tab cleanly via Decompose navigation — plugins can no longer bring down the host UI
- **Centralized .boss Data Directory**: `BossDirectories` centralizes all app data under `~/.boss` (or `~/.boss_debug` in dev mode) for consistent cross-platform data storage
- **BossEditor as Maven Artifact**: The local `bosseditor/` module has been removed; `com.risaboss:bosseditor-compose-desktop` is now resolved from Maven Central, simplifying the build and keeping the editor versioned independently
- **Thread-Safety in Crash Tracking**: Crash state uses `ConcurrentHashMap`, Compose state mutations are deferred to the EDT, and `closeAction`/exception handler factory fields are properly guarded

## Bug Fixes

- **plugins**: Fix `PluginCrashInterceptor` failing to attribute composition crashes to the correct plugin classloader
- **plugins**: Register crash interceptor eagerly and hook into Compose's uncaught exception handler so no crash escapes the sandbox
- **plugins**: Fix classloader attribution false positive causing unrelated errors to be treated as plugin crashes
- **plugins**: Guard `closeAction` in `recordCrash` and declare exception handler factory field as `volatile` to prevent data races
- **plugins**: Close crashed plugin tab via Decompose navigation instead of an inline fallback, ensuring proper lifecycle cleanup
- **plugins**: Force window repaint after deferred crash registry update so the fallback UI is visible immediately
- **plugins**: Fix plugin wizard always installing the latest version from the repository rather than a stale cached entry
- **window**: Fix window entering fullscreen animation on close (#649)
- **editor**: Remove dead RSyntax/PSI code, `CodeEditorSettings`, and rewire plugin API to `EditorSettingsManager`

## Improvements

- **plugins**: Add `PluginCrashRegistry` to force Compose subtree rebuild after a composition crash, with a styled fallback UI rendered above the corrupted `SubcomposeLayout` tree
- **plugins**: Defer `PluginCrashRegistry` state mutation to after the current render frame to avoid mid-composition state changes
- **system**: Introduce `BossDirectories` to centralize the `.boss` data directory; dev mode uses `~/.boss_debug`
- **editor**: Remove local `bosseditor/` module and consume `com.risaboss:bosseditor-compose-desktop` from Maven Central; add entry to version catalog

## Dependencies

- `io.github.jan-tennert.supabase` 3.3.0 → 3.4.1
- `com.fifesoft:rsyntaxtextarea` bumped to latest
