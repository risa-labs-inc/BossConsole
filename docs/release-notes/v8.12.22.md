# BOSS v8.12.22

**Release Date**: December 2024

## Summary

This release delivers comprehensive terminal rendering improvements and deep link handler fixes, resolving critical rendering glitches and URL handling issues when BOSS is set as the default browser.

## Highlights

- Fixed critical terminal blank display bug and empty line rendering issues that created massive empty areas
- Resolved deep link handler race conditions preventing external URLs from opening correctly
- Implemented iTerm2-style performance optimizations with O(1) buffer operations
- Added bundled Noto Color Emoji font for reliable emoji support across platforms

## Bug Fixes

### Terminal
- **Fixed terminal empty line rendering issues and improved performance** (#37)
  - Resolved critical blank black terminal area caused by malformed `getAnnotatedLines()` method
  - Fixed 138-space empty lines by filtering unnecessary space characters in grid rendering
  - Constrained text width to exact terminal dimensions to prevent text overflow
  - Implemented 5% PTY deception strategy to prevent Claude Code dialog overflow
  - Added cursor position mapping for PTY deception coordinate scaling
  - Fixed alternate screen buffer to not show scrollback inappropriately
  - Improved character handling logic in rendering pipeline
  - Added proper padding (12dp) and visual design improvements

### Browser
- **Resolved deep link handler not opening URLs correctly** (#116)
  - Fixed duplicate/empty window creation when URLs clicked from external apps (WhatsApp, email, etc.)
  - Implemented deferred URL handling system with proper initialization sequencing
  - Added URL queue to store URLs received during app initialization
  - Ensured URLs only processed after first window is fully created
  - Eliminated race conditions during startup when BOSS is set as default browser

## Improvements

### Terminal
- **Memory & Performance Optimization**
  - Implemented iTerm2-style lazy buffer allocation with single currentGrid pointer
  - Replaced O(n) scrollback operations with O(1) ArrayDeque operations
  - Added comprehensive `dispose()` method with proper memory cleanup
  - Fixed bounds validation for all cursor positioning operations
  - Removed all debug logging for production-ready codebase

- **Font & Color System**
  - Moved TerminalSettings from desktopMain to commonMain for universal access
  - Bundled Noto Color Emoji font (10MB) for reliable emoji support across platforms
  - Implemented iTerm2 Default color scheme with authentic 16-color ANSI palette
  - Created separate font options: "MesloLGS NF" (symbols) vs "MesloLGS NF + Emoji"
  - Added proper font fallback systems for Monaco, Menlo, JetBrains Mono, etc.
  - Resolved font symbol breaking when emoji fonts were globally applied

- **Technical Improvements**
  - Added Device Status Report (DSR) support for cursor position queries
  - Enhanced escape sequence handling with better error recovery
  - Fixed cursor glitches after alternate screen buffer operations
  - Fixed window sizing issues when Claude opens at bottom of screen
  - Improved terminal initialization and resize handling with proper debouncing
  - Fixed terminal component lifecycle to properly close panel when terminal stops
