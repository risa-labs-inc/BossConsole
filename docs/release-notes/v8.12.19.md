# BOSS v8.12.19

**Release Date**: October 2024

## Summary

This release adds comprehensive default browser functionality, allowing BOSS to be set as the system's default web browser for handling http/https URLs across all platforms.

## Highlights

- Cross-platform default browser support with native implementations for macOS, Windows, and Linux
- Automatic URL handling service that opens external links in Fluck browser tabs
- Settings UI for managing default browser status with real-time feedback
- Security-focused URL validation and routing

## Features

### Browser
- **Default browser functionality** (#112)
  - **macOS**: Automatic registration via Swift scripts using LSSetDefaultHandlerForURLScheme
    - Falls back to System Preferences if programmatic setting fails
    - Registered via Info.plist CFBundleURLTypes
  - **Windows**: Registry-based browser candidate registration
    - Creates keys in HKEY_CURRENT_USER\SOFTWARE\Clients\StartMenuInternet
    - Opens Settings app for manual selection (Windows 10+ requirement)
  - **Linux**: XDG standards compliance
    - Creates ~/.local/share/applications/boss.desktop file
    - Uses xdg-settings and xdg-mime for registration

### URL Handling
- **URL Handler Service** (#112)
  - Validates incoming http/https URLs for security
  - Creates Fluck browser tabs automatically
  - Extracts domain names for clean tab titles
  - Integrates with multi-window support
  - Rejects malformed URLs (missing domain, invalid format)

### Settings
- **Default Browser Settings UI** (#112)
  - Real-time default browser status indicator
  - "Set as Default Browser" button with loading states
  - Platform-specific instructions and messaging
  - Error handling with user-friendly dialogs
  - Refactored profile management into separate section for better organization

## Improvements

### Browser
- **Code organization** (#112)
  - Clean separation of concerns by platform (9 new files, ~1,410 lines)
  - All files kept under 300 lines for maintainability
  - Reusable UI components
  - Comprehensive error handling
  - Refactored FluckBrowserSettings.kt from 420 → 229 lines

### Code Quality
- **Removed dead code** (#112)
  - Removed unused `getDefaultHandlerForScheme()` function in macOS handler (26 lines)
  - Deleted unused `DefaultBrowserStatus` sealed class (17 lines)
  - Net change: -33 lines cleaner codebase

### Performance
- **Windows registry fail-fast logic** (#112)
  - Stops early on unrecoverable registry failures
  - Better performance for critical key operations
  - Returns immediately if main key, icon, or shell commands fail
