# BOSS v9.2.5

**Release Date:** 2026-06-02

Two related runner improvements: Claude/MCP-initiated sidebar runs now surface in the top-bar runner, and every run re-derives its working directory from the script's own project root at execution time.

## Highlights

- **MCP/Claude sidebar runs reflect in the top bar** (#763): a new `registerSidebarRun()` host entry (`RunnerTerminalService`) lets the terminal-tab `run_in_sidebar` MCP tool report back into the runner. A Claude-initiated sidebar run now adds its config to history, selects it in the window's dropdown, and shows as running (keyed to `SIDEBAR_TERMINAL_ID`, cleared by the existing tab-close callbacks) — without re-opening the tab the caller already created.
- **Run at the script's location** (#763): `resolveWorkingDirectory()` now re-derives a config's working directory from its source `filePath` (`findProjectRoot`) at execution time, so it always runs at the script's project root (the inner repo with `./gradlew`) instead of a stale or workspace-root path stored on a saved config. Applies to both the `cd`-prefixed command and the `workingDirectory` passed to the sidebar/main-panel terminal, on both run and re-run.

## Features

- **runner**: `registerSidebarRun()` — non-suspend host entry with primitive args that the `run_in_sidebar` MCP tool reflects into, so Claude-initiated sidebar runs appear in the top-bar runner (history + dropdown selection + running state). (#763)

## Improvements

- **runner**: `resolveWorkingDirectory()` re-derives the working directory from the config's source `filePath` at execution time, so runs always start at the script's project root rather than a stale or workspace-root path. (#763)
