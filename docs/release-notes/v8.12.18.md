# BOSS v8.12.18

**Release Date**: October 2025

## Summary

This release introduces comprehensive multi-window support, enabling users to create and manage multiple independent BOSS windows like standard desktop applications (Chrome, VS Code, etc.).

## Highlights

- Full multi-window support with independent window instances and lifecycle management
- Browser-standard keyboard shortcuts: Cmd+N (new window), Cmd+T (new tab), Cmd+W (close tab/window)
- Context menu "Open in New Window" option for tabs
- Event-driven window focus management with auth session integration
- GitHub API authentication support to prevent rate limiting during development

## Features

### Multi-Window Support
- **WindowManager for centralized lifecycle** (#109)
  - Tracks all open windows with SnapshotStateList
  - Cascade window positioning (30dp offset per window)
  - macOS-style app lifecycle (stays running when all windows closed)
  - Automatic window cleanup when empty

- **Independent window instances** (#109)
  - Each window gets isolated Decompose component context
  - Complete state and navigation isolation between windows
  - BossWindow composable wraps individual window instances
  - Cross-platform abstraction using expect/actual pattern

### Keyboard Shortcuts
- **Browser-standard shortcuts** (#109)
  - Cmd+N: Create new window
  - Cmd+Shift+N: Create new window (explicit)
  - Cmd+T: Show new tab dialog (URL/Editor/Terminal selection)
  - Cmd+W: Close tab (auto-closes window if last tab)
  - Cmd+Shift+W: Force close window
  - All shortcuts handled at Compose level for reliable event handling

### UI Enhancements
- **Tab context menu** (#109)
  - "Open in New Window" option for moving tabs to dedicated windows
  - Integration with WindowManager for seamless tab migration

### Focus Management
- **Event-driven focus system** (#109)
  - Window focus triggered by auth session resolution (300-800ms typical)
  - Replaces hard-coded delays with reactive StateFlow collection
  - Reliable across different network conditions
  - New windows automatically receive focus for immediate keyboard response

## Improvements

### Developer Experience
- **GitHub API authentication** (#109)
  - Configurable GitHub token support via environment, system properties, or local.properties
  - Automatic GitHub CLI token detection (`gh auth token`)
  - Increases rate limit from 60/hour (unauthenticated) to 5,000/hour (authenticated)
  - Better error handling for rate limits and API failures
  - Documented in CLAUDE.md with configuration examples

### Code Organization
- **Cross-platform architecture** (#109)
  - WindowOperations.kt: Platform-independent interface
  - DesktopWindowOperations.kt: Desktop implementation
  - GitHubConfig.kt: Centralized GitHub token configuration (70 lines)
  - WindowManager.kt: Window lifecycle management (181 lines)
  - BossWindow.kt: Individual window wrapper (71 lines)

### Performance
- **Faster window focus** (#109)
  - Event-driven approach: 300-800ms vs previous 600-1500ms delays
  - No arbitrary waits or retry loops
  - Focus only requested when auth session fully resolved

## Bug Fixes

### Window Lifecycle
- **Fixed immediate exit bug** (#109)
  - Create initial window BEFORE application scope to prevent empty window list
  - Ensures at least one window renders on first frame

- **Fixed focus conflicts** (#109)
  - Use Compose key() to prevent window re-composition when new windows added
  - Prevents LaunchedEffect from running multiple times on existing windows
  - Eliminated race conditions between windows competing for focus

### Keyboard Shortcuts
- **Fixed duplicate event handlers** (#109)
  - Moved all shortcuts from AWT to Compose level
  - Compose's onPreviewKeyEvent intercepts BEFORE AWT handlers
  - Removed dead AWT registerKeyboardAction code that never fired
  - Fixed Cmd+N opening tab dialog instead of creating window
  - Fixed Cmd+T opening terminal instead of showing tab selection

### Build System
- **Fixed test compilation** (#109)
  - Added JUnit 5 dependencies for desktop tests
  - Added kotlin-test-junit5 and junit-jupiter dependencies
  - Resolved compilation errors in UpdateScriptGeneratorSecurityTest.kt
