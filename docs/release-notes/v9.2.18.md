# BOSS v9.2.18

**Release Date:** 2026-06-30

Introduces the "Operator's Console" design system — a unified set of design tokens shared by the host and terminal that makes the entire app re-skin live, plus a Settings theme picker (Operator / Daylight / Clean) and browser content that follows the active host theme.

## Highlights

- **Operator's Console design system**: A single source of truth for palette, spacing, radius, elevation, motion, and typography (`BossDesignSystem.kt`), exposed via the `BossTheme` accessor. `BossColors` delegates to `BossPalette` so existing components re-skin to the amber signal automatically, with MesloLGS wired as the mono brand voice (#808).
- **Theme-aware host, end-to-end**: Reactive theme tokens are adopted across the host so the whole app re-skins live on theme switch, with a Settings theme picker offering three selectable themes — Operator, Daylight, and Clean — persisted to `~/.boss` and applied at startup (#808).
- **Browser follows the host theme**: `FluckEngine` mirrors the active BOSS theme into the live JxBrowser engine via `Engine.setTheme(LIGHT/DARK)`, so web content's `prefers-color-scheme` matches the app (Daylight → light; Operator/Clean → dark) (#808).

## Features

- **panel**: Add the "Operator's Console" design system — unified design tokens (palette, spacing, radius, elevation, motion, typography) in `BossDesignSystem.kt` exposed through the `BossTheme` accessor, with `BossColors` delegating to `BossPalette` and MesloLGS as the mono brand voice. Migrates `BossTabButton`, `BossActionButton`, `ConfirmationDialog`, and `ContextMenu` to semantic tokens. Adds three selectable themes (Operator / Daylight / Clean) plus light/clean color schemes, a reactive `BossTheme` re-export, and a Settings theme picker persisted to `~/.boss` and applied at startup; ~38 dialogs, settings sections, the panel title bar, and the tab-bar "+" button were migrated to theme tokens. Public `BossTheme(content)` JVM signature is preserved for plugin binary compatibility. Adds `docs/DESIGN_SYSTEM.md` (#808)

## Bug Fixes

- **browser**: Fix `FluckEngine` initialization order — `startHostThemeObserver()` ran during object init while `themeScope` was still null, causing an `ExceptionInInitializerError`/NPE on startup; field declarations are now moved above the `init {}` block (#808)

## Improvements

- **browser**: `FluckEngine` observes `BossThemeController` and applies the host theme to the running engine live, re-applying it on engine (re)creation and remaining safe before the engine exists (#808)
