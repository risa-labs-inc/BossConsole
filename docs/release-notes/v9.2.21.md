# BOSS v9.2.21

**Release Date:** 2026-07-02

A performance-focused release: startup no longer walks the entire disk, file scans and plugin loading move off the UI thread, and connectivity probes race for first success. Also fixes split-divider dragging so it tracks the live container size and enforces a minimum pane size.

## Highlights

- **No more whole-disk indexing at startup**: A Finder-launched packaged app reported `user.dir` as `/`, so the editor indexer walked the entire disk at ~100% CPU for the whole session. Startup no longer indexes `user.dir` — editors index the real project root when they open (#817).
- **Snappier, non-blocking startup**: PSI warm-up, app-update Realtime, plugin JAR validation/classloading, and bundled-plugin/store disk work all move off the main thread, and connectivity probes now race primary + fallback for first success (worst case was 2×10 s before the first usable screen) (#817).
- **Correct split-divider dragging**: Dividers now resize against the live container size with a stored weight fraction, so drags no longer reset on unrelated recompositions; both sides keep at least 10% of the container (#816).
- **Faster file scanning**: `hasChildren` now streams entries and stops at the first visible one instead of materializing the whole listing, and directory scans run on `Dispatchers.IO` so plugin callers no longer block the UI (#815).

## Bug Fixes

- **splits**: `BossResizablePanel` derived the split fraction from a non-remembered local that stuck at a stale container size, so dragging moved the divider by a `newSize/oldSize` multiplier that silently reset on unrelated recompositions and clamped min/max to first-composition window size. Relative panels now store a weight fraction applied against the live container size, the gesture reads bounds via `rememberUpdatedState`, and both sides keep at least 10% so sidebars can no longer swallow the main content (#816)
- **network**: First-success-wins connectivity race — `coroutineScope` previously waited for both probes because the blocking `HttpURLConnection` calls ignore cancellation, stalling the connected path for the full timeout when the fallback host was unreachable; the all-failed path is now driven from `invokeOnCompletion` so `await()` can't hang if the service scope is cancelled mid-race (#817)
- **file-scanner**: Catch `InvalidPathException` in `directoryHasChildren` — `Paths.get(path)` can throw for malformed input (e.g. a NUL byte) reachable from plugins via `FileSystemDataProvider`, so it now degrades to `false` instead of propagating (#815)

## Improvements

- **startup**: PSI warm-up and app-update Realtime start moved off the main thread; bundled-plugin copies, store downloads, and JAR reconcile run on `Dispatchers.IO` with startup timing logs; `DynamicPluginLoader` runs JAR validation/classloading/instantiation on IO. Fire-and-forget warm-ups now share one named process-lifetime supervised scope (#817)
- **file-scanner**: `directoryHasChildren` streams entries and stops at the first visible one instead of materializing the full listing via `listFiles`; `scanFileRecursively` stats each entry once before sorting (the old `compareBy` comparator re-invoked `File.isDirectory` on every comparison) and no longer re-scans a directory it already listed; `scanDirectory`/`scanDirectoryWithDepth` run on `Dispatchers.IO`. Adds `DesktopFileScannerTest` covering empty dirs, filtered-only dirs, symlinks, and the malformed-path regression (#815)
- **theme**: `BossTheme` now remembers the typography — a fresh `FontFamily` instance per recomposition changed the static composition local's value and recomposed the entire subtree under the theme root (#817)

## Dependencies

- **editor**: Bump BossEditor to 1.0.4 (indexer walk + unsafe-root fixes) (#817)
