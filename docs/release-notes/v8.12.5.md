# BOSS v8.12.5

**Release Date**: October 2024

## Summary

This release introduces a comprehensive Admin Role Management plugin that enables administrators to manage user roles through a powerful UI with search, pagination, role assignment/removal, and user deletion capabilities.

## Highlights

- New Admin Role Management plugin with full user management interface
- Server-side search and pagination for efficient handling of large user bases
- Role assignment/removal with UI and database-level protections
- User deletion functionality with confirmation dialogs (non-admin users only)
- Dynamic panel visibility based on admin status with proper lifecycle management

## Features

### Admin Role Management Plugin (#71)

**Core Functionality:**
- View all users with their assigned roles in a clean, organized interface
- Search users by email with server-side filtering (searches entire database, not just loaded users)
- Instagram-style pagination for efficient loading (50 users per page)
- Assign and remove roles with real-time updates
- Delete user accounts with confirmation dialogs
- Visual role badges with shield icons for admin roles

**User Interface:**
- Clean search bar with real-time filtering
- Role badges with color coding (unified blue theme with admin shield icons)
- Delete buttons with trash icons (hidden for admin users)
- Loading states and success/error notifications
- "Load more" functionality with automatic detection
- Access denied screen for non-admin users

**Security Features (Defense in Depth):**
- All operations protected by Row Level Security (RLS) policies
- Cannot delete admin users (must remove admin role first)
- Cannot delete yourself
- Cannot remove your own admin role
- USER role is non-removable (baseline role protection)
- Server-side validation with SECURITY DEFINER functions

**Performance Optimizations:**
- Server-side search with PostgreSQL ILIKE (case-insensitive)
- Pagination reduces initial load time from 10+ seconds to 1-2 seconds
- Memory-efficient: only loaded pages kept in memory
- Database indexes for fast search queries

**Database Functions:**
- `delete_user()` - Admin-only function to delete user accounts with validations
- Updated `remove_role_from_user()` - Prevents removal of baseline 'user' role

## Bug Fixes

### Authentication (#71)

- **Fixed admin panel visibility on app restart** - SessionManager now properly parses role claims from JWT when loading from storage, ensuring admin panel appears correctly after restart

### Security (#71)

- **Fixed infinite recursion in RLS policies** - Replaced database queries with JWT claims checks to prevent circular dependencies in user_roles and role_permissions tables
- **Fixed permission denied errors** - Added explicit GRANT SELECT permissions on RBAC tables for authenticated role
- **Fixed serialization issues** - Added @SerialName annotations for proper snake_case to camelCase mapping between database and Kotlin

### Memory Leaks (#71)

- **Fixed GlobalScope memory leak** - Replaced GlobalScope.launch() with lifecycle-aware pluginScope in panel registration
- **Fixed ViewModel coroutine scope leak** - Added proper dispose() method and lifecycle integration
- **Fixed race conditions** - Used distinctUntilChanged() to prevent race conditions in dynamic panel registration

## Improvements

### Database (#71)

- **JWT-based RLS policies** - All RBAC table policies now use JWT claims instead of database queries, eliminating circular dependencies
- **Enhanced delete_user() function** - Comprehensive validations prevent deleting admins and yourself
- **Role removal protection** - Database-level enforcement prevents removal of baseline 'user' role

### UI/UX (#71)

- **Dynamic panel registration** - Admin Role Management panel automatically appears/disappears based on auth state
- **Improved role badges** - Unified blue theme with shield icons for admin roles, scalable for future plugin-specific roles
- **Better button icons** - Changed remove icon from minus to X for clearer action indication
- **Pagination indicators** - Shows loading state and "No more users" message when all users loaded

### Architecture (#71)

- **Proper lifecycle management** - All coroutines tied to component lifecycle with automatic cleanup
- **Change listener mechanism** - PanelRegistry supports dynamic updates for reactive panel registration
- **Decompose integration** - Used lifecycle.doOnDestroy() for proper resource cleanup
