# BOSS v8.16.28

**Release Date:** 2026-02-08

Stable release introducing mandatory tab plugins for streamlined setup and improved Windows CI/CD reliability.

## Highlights

- **Mandatory Tab Plugins**: Plugin wizard now includes three essential plugins that cannot be deselected
- **GitHub Integration**: Direct GitHub release downloads with optional token support for higher rate limits
- **Windows Build Stability**: Resolved Kotlin daemon file handle issues on Windows CI

## Features

- **wizard**: Add mandatory tab plugins to plugin wizard (#613)
  - Include three essential plugins that ship with every installation:
    - Browser Tab (fluckbrowser) from GitHub
    - Code Editor Tab (editortab) from GitHub
    - Terminal Tab (terminaltab) from GitHub
  - Add isMandatory flag to prevent deselection of core plugins
  - Add GitHub release download support with asset discovery
  - Support GitHub token configuration for higher API rate limits (5000 vs 60 req/hr)
  - Add manifest ID verification with warning logging
  - Filter out test artifacts (-test JARs) from downloads

## Bug Fixes

- **ci**: Fix Windows build failure due to Kotlin daemon file handle issues (333cf9f4)
  - Add Windows-specific Gradle options to disable daemon and parallel builds
  - Stop lingering Kotlin daemons before Windows builds to prevent file contention
  - Fix "The handle is invalid" error during IR lowering on Windows
  - Reference: KT-55959 (Kotlin file handle issues on Windows CI)

## Improvements

- **wizard**: Fix HTTP connection leaks by properly calling disconnect() in finally blocks
- **plugin-manager**: Add debug logging for manifest extraction with apiVersion/minBossVersion
- **wizard**: Add manifest verification to ensure plugin IDs match expected values
