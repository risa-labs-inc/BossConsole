# BOSS v9.2.40

**Release Date:** 2026-07-10

Fixes moving a live tab between split panels: the tab's real component now transfers intact instead of being recreated from config, so browser tabs no longer reload — and the old instance no longer leaks a live Chromium process (YouTube audio kept playing from the invisible leaked instance) (#853).

## Highlights

- **Cross-panel tab moves transfer the live component**: `MoveToPanel` and cross-panel `CreateSplit` drops now detach the live component from the source panel and let the target panel adopt it as-is — no reload, media keeps playing in the moved tab, and nothing is left running behind. Previously the move destroyed and recreated the component from config, reloading browser tabs from their URL while the old browser instance kept running invisibly (#853).
- **Per-tab lifecycle stops leaked Chromium processes**: Each tab component now gets its own `LifecycleRegistry` (resumed on add, destroyed on remove) instead of sharing its panel's never-destroyed one, so a dynamic plugin tab's `lifecycle.onDestroy` cleanup (e.g. fluck-browser disposing its JxBrowser handle) actually fires. Closing *or* moving a browser tab no longer leaks a live Chromium process (#853).

## Bug Fixes

- **tabs**: move live tab components across splits instead of recreate-and-leak — cross-panel `MoveToPanel`/`CreateSplit` drops now transfer the live component (browser tabs no longer reload, media keeps playing) rather than rebuilding it from config, and each tab gets its own destroyable `LifecycleRegistry` so `onDestroy` cleanup fires and no leaked Chromium process is left running invisibly. `TabUpdateRegistry.createProvider` now re-resolves the tab's owning panel on every call so a cached provider (the browser wires its navigation/title listeners against it once) keeps updating the tab bar after a move; same-panel edge drops and context-menu Split Right/Down keep their existing copy semantics (#853)
