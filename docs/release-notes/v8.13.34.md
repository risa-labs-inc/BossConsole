# BOSS v8.13.34

**Release Date**: December 2025

## Summary

This release brings significant improvements to browser stability and terminal functionality, with robust recovery mechanisms for JxBrowser issues, configurable recovery settings, and support for opening terminals in project-specific working directories.

## Highlights

- Automatic browser validity checks and recovery for closed/crashed browsers
- Configurable browser retry and recovery limits via Settings
- Terminal now opens in project folder when a project is selected
- Prevention of Supabase request cancellation errors

## Features

### Browser
- **Configurable browser retry and recovery limits** (#354)
  - Added Settings UI for controlling browser recovery behavior
  - Allows users to customize recovery attempts and limits
- **User-friendly error handling for JxBrowser license validation failures** (#354)
  - Clear error messages when license validation fails
  - Improved troubleshooting experience
- **Event-driven browser closure detection** (#354)
  - Replaced polling mechanism with efficient event-driven approach
  - Better resource utilization
- **Recovery UI message on browser issues** (#354)
  - Visual feedback when browser recovery is in progress
  - Validates browser health on tab switch

### Terminal
- **Terminal opens in project folder when selected** (#350)
  - Added workingDirectory support to sidebar terminal
  - Uses native workingDirectory API from BossTerm 1.0.55
  - Terminal automatically opens in the correct project context

## Bug Fixes

### Browser
- **Browser validity check to detect and recover from closed browsers** (#351, #354)
  - Detects when JxBrowser is closed or crashed
  - Automatically attempts recovery
  - Skips recovery when tab is intentionally disposed
- **Auto-reload browser tabs when FluckEngine reinitializes** (#351, #353)
  - Browser tabs automatically reload after engine recovery
  - Prevents blank or unresponsive browser tabs
- **Thread safety and recovery loop prevention** (#354)
  - Added synchronization to prevent race conditions
  - Prevents infinite recovery loops

### API
- **Prevent Supabase request cancellation errors** (#352)
  - Fixed premature cancellation of Supabase API requests
  - Improved reliability of backend communication

## Improvements

### Browser
- **Extract duplicate browser recovery logic into resetForRecovery()** (#354)
  - Consolidated recovery logic into reusable method
  - Improved code maintainability and consistency

## Dependencies

- **BossTerm updated to 1.0.56** (#356)
  - Includes native workingDirectory API support
  - Enhanced terminal capabilities
