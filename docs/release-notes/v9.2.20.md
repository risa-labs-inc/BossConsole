# BOSS v9.2.20

**Release Date:** 2026-07-02

Introduces a plugin MCP tool framework so any dynamic plugin can contribute `mcp__boss__*` tools that appear while it's active and are gated by RBAC, and fixes release publishing of large assets to Supabase Storage via TUS resumable uploads.

## Highlights

- **Plugin MCP tool framework**: Any dynamic plugin can now contribute `mcp__boss__*` tools that appear while the plugin is active and disappear on disable/unload, with per-tool `requiredPermissions`/`requiresAdmin` gating that follows role changes live (#814).
- **Large release assets upload reliably**: Assets over 50 MB now upload through Supabase's TUS resumable endpoint in 6 MiB chunks, so no single request hits Cloudflare's ~100 MB body cap that previously returned HTTP 413 for BOSS's `.dmg` (#813).

## Features

- **plugins**: Plugin MCP tool framework — new `McpTool` API (`McpToolProvider`/`Definition`/`Handler`/`Args`/`Result`, `McpToolRegistry`) and `McpServerController`; `PluginContext` gains `registerMcpToolProvider`/`unregisterMcpToolProvider`/`mcpToolRegistry`. Exposed tools = registered − user-disabled − permission-denied, with the disabled set persisted to `mcp-disabled-tools.json`. `DynamicPluginManager` pushes `isAdmin`/`userPermissions` into the registry so permission-gated tools follow role changes live. Pairs with boss-plugin-api 1.0.51 and terminal-tab 2.5.0; adopted by 19 plugins (#814)

## Bug Fixes

- **release**: Upload large assets (> 50 MB) via Supabase TUS resumable uploads instead of a single `curl POST`, fixing the HTTP 413 Cloudflare returns for BOSS's oversized `.dmg` before Supabase sees the request; smaller assets keep the single-POST path (#813)
- **release**: Bounded retry with backoff around resumable create + each `PATCH` chunk, re-reading the true server offset via `HEAD` so a transient blip resumes rather than aborting the whole transfer (#813)

## Improvements

- **plugins**: Serialize all `McpToolRegistryImpl` mutators and recompute under a single lock (preventing a stale snapshot from leaving an unloaded plugin's tools invocable or a revoked tool exposed), persist the disabled set atomically via temp-file+rename, and tear down partial registrations when `register()` throws (#814)
- **plugins**: Extract testable `McpToolRegistryCore` behind the singleton and add 27 unit tests covering permission semantics, first-wins dedup, disabled-set persistence, argument coercion, and the invoke timeout/cancellation contract (#814)

## Dependencies

- **ci**: Bump `actions/checkout` from 4 to 7 (#812)
- **ci**: Bump `supabase/setup-cli` from 1 to 2 (#811)
