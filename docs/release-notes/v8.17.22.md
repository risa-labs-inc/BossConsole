# BOSS v8.17.22

**Release Date:** 2026-03-12

This release completes the split-brain microkernel architecture with full out-of-process plugin isolation, 15 new IPC service bridges, and parallel OOP child process spawning that reduces plugin load time from ~11s to ~1s — along with comprehensive bug fixes for thread safety, gRPC registration reliability, and plugin restart logic.

## Highlights

- **Plugin OOP isolation**: Full split-brain microkernel implementation with 15 proto service definitions, 15 kernel-side gRPC bridges, and 19 IPC proxy wires in `RemotePluginContext`
- **16x faster plugin startup**: OOP child processes now spawn in parallel via background coroutines — all 16 plugins load in ~1s instead of ~11s
- **Microkernel Mode UI**: File menu checkbox and Advanced Settings toggle for switching between MICROKERNEL and MONOLITH modes; `BOSS_MODE` now reads from `local.properties`/env vars via `ConfigLoader`
- **Configurable plugin heap**: Per-plugin JVM heap (128MB–8GB) in Settings > Advanced, with smart default of 2% of system RAM auto-detected

## Features

- **microkernel**: Complete out-of-process plugin isolation — `RemotePluginContext` wires 19 IPC proxies, `OutOfProcessPluginSpawnerImpl` uses `BossDirectories`-aware paths, `BridgedPluginViewModel` + `BridgedPanelComponent` for kernel-side UI rendering
- **microkernel**: 15 new gRPC service bridges (performance, download, git, log, context menu, plugin state, etc.) registered via `KernelBootstrap.registerPluginServices()`
- **settings**: Microkernel Mode checkbox added to File menu and Advanced Settings panel
- **config**: `BOSS_MODE` now read from `ConfigLoader` (supports `local.properties` and environment variables), not just `System.getenv`

## Bug Fixes

- **microkernel(grpc)**: Fix `BossIpcServer` silently dropping all 15 plugin gRPC bridges registered after `start()` — now rebuilds server when services are added post-start
- **microkernel(grpc)**: Replace fragile reflection-based `KernelBootstrap` access in `waitForReady()` and `buildJvmArgs()` with direct instance access to avoid breakage under obfuscation
- **microkernel(server)**: Fix `BossIpcServer.rebuildServer()` to stop old server before starting new one, preventing "address already in use" on Unix domain sockets
- **microkernel(plugins)**: Fix `PluginProcessMonitor.restartPlugin()` to store manifest and jar path at `monitor()` time and call `spawner.spawn()` after `terminate()`, so the restart button actually re-spawns the child JVM
- **microkernel(plugins)**: Fix split-brain plugin registration using the already-loaded plugin instance instead of calling `loadPlugin()` a second time, which caused silent failure and missing sidebar panels
- **microkernel(plugins)**: Fix Processes tab to use `getProcessesByType(PLUGIN)` against the kernel registry instead of broken string-comparison filtering
- **microkernel(state)**: Fix delta state sync to fall back to full state fetch instead of corrupting state by applying raw patch bytes as replacement
- **threading**: Use `AtomicLong` for `PluginStateHolder._version`, `CopyOnWriteArrayList` for `ConsoleStateHolder.allLogs` (capped at 10k entries), `ConcurrentHashMap.newKeySet()` for `PluginProcessMonitor.inProcessFallbacks`, and `AtomicInteger` countdown in `AdminStateHolder` to prevent premature `isLoading=false`
- **i18n**: Add `Charsets.UTF_8` to env_vars file I/O in `BossWindow.kt` and `AdvancedSettings.kt` to prevent corruption on Windows

## Improvements

- **microkernel(perf)**: Spawn OOP child processes in parallel via `managerScope.launch`; plugin UI registered immediately in kernel while JVM boots in background
- **microkernel(perf)**: Cache child process metrics for 5s to reduce `ps` subprocess spawning ~60x
- **microkernel(perf)**: Reduce intent/effect gRPC buffer sizes (256→32, 64→16) and add jittered exponential backoff to `PluginStateBridge` reconnection
- **microkernel(perf)**: Add JVM shutdown hook to kill orphaned child processes on exit or crash
- **microkernel(settings)**: Configurable plugin JVM heap (128MB–8GB) in Settings > Advanced with smart 2%-of-RAM default
- **microkernel(monitoring)**: Fix Performance Processes tab to use `ProcessHandle` API and `ps` for real RSS memory, thread count, and uptime metrics instead of a broken reflection chain
- **ai**: Update `RepairEngine` to use GPT-5.4 with `OPENAI_API_KEY` fallback for AI self-healing
- **ci**: Pin `actions/setup-java` to `v4` to resolve 401 Unauthorized download errors on GitHub runners
