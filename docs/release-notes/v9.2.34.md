# BOSS v9.2.34

**Release Date:** 2026-07-06

Adds `PanelEventProvider.openPanel(panelId, windowId)` so a plugin can reveal another plugin's sidebar panel — the same reveal path `boss://plugin?id=` uses — wired end-to-end including over IPC for out-of-process plugins.

## Highlights

- **Plugins can open another plugin's sidebar panel**: A new default-no-op `openPanel(panelId, windowId)` on the plugin-api-core `PanelEventProvider` is implemented in `PanelEventProviderImpl` via `PanelEventBus.openPanel`, letting one plugin reveal another's panel — used by the Toolbox "Create" tab to launch Tool Creator. Pairs with boss-plugin-api 1.0.57 (#844).
- **Works for out-of-process plugins too**: `openPanel` is forwarded over gRPC (new `OpenPanel` RPC + `OpenPanelRequest` in `panel_event.proto`, an override in `PanelEventProviderProxy`, and handling in `PanelEventServiceBridge`), mirroring the existing `closePanel` wiring — without it a kernel-mode plugin's `openPanel` fell through to the interface default no-op and silently did nothing (#844).

## Features

- **panel**: `PanelEventProvider.openPanel(panelId, windowId)` reveals a sidebar panel from a plugin, and is forwarded over IPC so out-of-process plugins can trigger it too (#844)
