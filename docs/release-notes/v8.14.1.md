# BOSS v8.14.1

**Release Date**: December 2025

## Summary

This release improves terminal integration with configurable link prompts, adds a Reset Terminal feature, and enhances the New Tab Dialog with file browsing capabilities. Multiple bug fixes improve browser error handling and session restoration.

## Highlights

- **Terminal link prompt** - Click prompts for terminal links with "remember preference" option (#346)
- **Reset Terminal feature** - New Help menu option and panel action to reset terminal state
- **Enhanced file browsing** - New Tab Dialog now includes file tree browser and folder selection
- **Browser error improvements** - Scrollable error pages and proper state management (#358)

## Features

### Terminal
- **Terminal link click prompt** (#346, #366, #367)
  - Added configurable prompt when clicking terminal links
  - "Remember preference" option to skip future prompts
  - Improved status bar feedback for terminal operations
  - Extracted RUNNER_TERMINAL_PREFIX constant for consistency

- **Reset Terminal** (#365)
  - New "Reset Terminal" option in Help menu
  - Panel "more menu" action for quick terminal reset
  - Bound to panel context menu for easy access

- **BossTerm updates**
  - Updated BossTerm to 1.0.63 with onTabClose handler support
  - Updated to BossTerm 1.0.62 for improved terminal management
  - Proper cleanup of runner state when terminal tabs are closed

### UI
- **Enhanced New Tab Dialog** (#367)
  - File tree browser for visual file selection
  - Folder selector dropdown for navigation
  - File browse button for quick access
  - Empty folder state indicator
  - Removed default file path for cleaner UX

- **Workspace improvements**
  - Configurable startup timeout setting
  - Briefcase icon for workspace button
  - Fixed workspace icon display

## Bug Fixes

### Browser
- **Error page improvements** (#358, #363)
  - Made browser error page scrollable
  - Fixed browser error state persistence
  - Proper engine state reset on "Reset Tab" button click

### UI
- **Session restoration fixes**
  - Prevented NewTabDialog from appearing during session restoration
  - Fixed LaunchedEffect restart when startup settings load
  - Improved state management during app initialization

### Terminal
- **Runner state cleanup**
  - Clean up runner state when sidebar terminal tab is closed
  - Clean up runner state when terminal tab is closed directly
  - Proper handler registration with BossTerm updates
