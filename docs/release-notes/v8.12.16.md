# BOSS v8.12.16

**Release Date**: October 2024

## Summary

This release fixes the critical macOS update installation failure caused by file locks, implements proper helper script patterns for seamless updates, and hardens update script security against command injection vulnerabilities.

## Highlights

- Fixed macOS update installation failures caused by apps trying to delete themselves while running
- Implemented industry-standard helper script pattern for graceful app replacement
- Critical security hardening for update scripts with comprehensive input validation and shell escaping
- Improved magic link error handling with clear user feedback for expired/invalid links

## Bug Fixes

### Update Installation
- **Fixed macOS update installation failure** (#107, Issue #104)
  - Resolved "permission denied" errors when updating on macOS
  - Problem: App tried to delete itself while running (impossible due to file locks)
  - Solution: Helper script waits for app to quit, then installs update
  - New flow: Generate script → Quit app → Script waits for PID → Install → Launch → Self-destruct
  - Created UpdateScriptGenerator.kt (260 lines) for platform-specific helper scripts
  - Created UpdateInstaller.kt (302 lines) with InstallResult sealed class
  - Refactored DesktopUpdateService.kt (476 → 257 lines) to delegate installation logic
  - Added quitForUpdate() to ApplicationRestarter.kt for clean shutdown without restart

- **Fixed race condition in quitForUpdate()** (#107)
  - Replaced fire-and-forget GlobalScope.launch with runBlocking
  - Ensures cleanup executes synchronously before shutdown
  - Prevents race conditions where other code could execute during cleanup window

- **Fixed DMG cleanup robustness** (#107)
  - Added try-finally block to ensure DMG is always unmounted on macOS
  - Prevents resource leaks and stale mounts when errors occur

### Authentication
- **User feedback for expired/old magic links** (#107, Issue #101)
  - Fixed silent failures when clicking expired or already-used magic links
  - Created MagicLinkErrorService.kt for centralized error state management
  - Clear error messages displayed above "Resend Magic Link" button
  - Example messages:
    - "This magic link has expired. Magic links are valid for 15 minutes..."
    - "This magic link has already been used..."
    - "Network request cancelled. Please check your internet connection."
  - Errors automatically clear on navigation

## Security

### Update Script Hardening
- **Fixed critical command injection vulnerability** (#107)
  - Attack vector: Malicious filenames from compromised GitHub releases or MITM attacks
  - Example: `BOSS-8.12.14$(rm -rf ~/Documents).dmg` would execute arbitrary commands
  - **Shell argument escaping** - Added escapeShellArg() using single quotes to prevent ALL interpolation
  - **Windows batch escaping** - Added escapeWindowsArg() to properly escape quotes and percent signs
  - **Path validation** - Rejects null bytes, shell metacharacters ($, `), newlines, path traversal (..)
  - **Early file validation** - Validates existence, extension, canonicalized paths before installation
  - Defense-in-depth: Input validation → Shell escaping → Logging → Testing
  - Created UpdateScriptGeneratorSecurityTest.kt with comprehensive security tests

- **Hardened path validation to fail-fast** (#107)
  - Path traversal (..) now rejected with SecurityException (was warning)
  - Command separators (;, |, &) now rejected (was warning)
  - Windows metacharacters (%, ^, !) now rejected
  - Legitimate update paths never use these characters
