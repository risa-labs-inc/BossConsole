# BOSS v9.2.2

**Release Date:** 2026-05-30

IPC compatibility hardening: per-version `min_ipc_version` is now plumbed end-to-end from the plugin store through `PluginUpdateManager`, the host's own system-plugin auto-updater is gated by `IpcVersion.CURRENT`, and an IPC version drift between `IpcVersion.CURRENT="1.0.0"` and `:boss-ipc` module `1.1.0` is corrected so release builds pass the `verifyIpcVersionConsistency` check.

## Highlights

- **Plugin store per-version IPC gating** (#747): `plugin_versions.min_ipc_version` is now part of the schema, edge-function payloads, and the client `VersionInfo` / `DownloadInfoResponse` / `PluginInfo` models. `PluginUpdateManager` takes an injected host-IPC predicate, offers only compatible newer versions, and reports the rest as `IncompatibleNotice` rather than installing them.
- **Host system-plugin auto-update gate generalized** (#746): `downloadSystemPluginFromGitHub` previously only checked IPC compat for the microkernel runtime — and the gate's reflection was a silent no-op because `IpcVersion.CURRENT` is a `const val` (static field, not `getCURRENT()`). Both are fixed: the gate runs for every system plugin (terminal-tab, microkernel runtime, …), and reads the static `CURRENT` field directly. Pinned by `PluginStoreSetupIpcGateTest`.
- **`IpcVersion.CURRENT` rolled back to 1.0.0** (#745, #749): PR #742 pivoted terminal-tab back to in-process, so the host no longer implements any v1.1.0 streaming RPCs — `TerminalServiceImpl` only exposes the v1.0.0 surface. `CURRENT` now correctly advertises `1.0.0`, the `:boss-ipc` module version is also rolled to `1.0.0` to silence `verifyIpcVersionConsistency`, and the v1.1.0 proto scaffolding stays in `terminal.proto` (forward-compatible, cheap to keep).
- **Sandbox health metrics race fixed** (#744): `_healthMetrics.value` was read-modify-write, so the heartbeat coroutine could clobber `recordError`/`recordSuccess` mutations under kotlinx-coroutines 1.11.0's scheduler — `consecutiveErrors` drops flaked `HealthMetricsTests`. Switched to atomic `MutableStateFlow.update { }`.

## Features

- **plugin-store**: Per-version `min_ipc_version` recorded server-side and surfaced in `VersionInfo` / `DownloadInfoResponse` / `PluginInfo`; `PluginRepositoryManager.getPluginVersions` exposes the full version list so the plugin-manager plugin can render badges and offer downgrades. No new endpoints or SQL semver comparison — the server reports, the host (which owns `IpcVersion.CURRENT`) decides. (#747)
- **plugin-updater**: `PluginUpdateManager.checkForUpdates` routes incompatible newer versions to a new `IncompatibleNotice` outcome instead of installing them; `IpcCompatibility` wraps `IpcVersion` by reflection (boss-ipc is excluded on Windows-ARM64) and is injected from `PluginStoreSetup`. Pinned by `PluginUpdateManagerCheckCompatTest` and `IpcCompatibilityTest`. (#747)

## Bug Fixes

- **plugin-loader**: `downloadSystemPluginFromGitHub`'s IPC-compat gate is no longer microkernel-only and no longer a silent no-op — incompatible downloaded JARs are discarded and the existing compatible JAR is kept until BossConsole itself updates. Legacy JARs with a blank `minIpcVersion` are unaffected. Extracted into `PluginStoreSetup.ipcIncompatibilityReason()` with a regression test pinning the boss-ipc reflection contract. (#746)
- **ipc**: `IpcVersion.CURRENT` rolled back to `"1.0.0"` to match what `TerminalServiceImpl` actually implements — the v1.1.0 streaming surface was retired with terminal-tab's in-process pivot (#742). v1.1.0 messages and RPC declarations are kept in `services/terminal.proto` so a future minor bump can re-enable streaming without re-running the proto changes. (#745)
- **build**: `:boss-ipc` Gradle module version rolled from `1.1.0` to `1.0.0` so it tracks `IpcVersion.CURRENT` — fixes `verifyIpcVersionConsistency` failing the Release Build with `IPC version drift: IpcVersion.CURRENT='1.0.0' but :boss-ipc.version='1.1.0'`. (#749)
- **sandbox**: `_healthMetrics.value = ...` replaced with `_healthMetrics.update { }` so the heartbeat coroutine can't lose interleaved `recordError`/`recordSuccess` mutations; un-flakes `HealthMetricsTests` under kotlinx-coroutines 1.11.0's scheduler. (#744)
