# BOSS v9.1.4

**Release Date:** 2026-05-12

Bug-fix release that preserves the in-flight POST body when a form-submit popup (e.g. OncoEMR's Print → `target="_blank"`) is adopted as a BOSS tab, so the new tab replays the original POST instead of issuing an empty GET.

## Highlights

- **Form-submit popups now retain their POST body**: BOSS captures the upload data of the popup's pending navigation and replays it on the adopted tab's first load, fixing flows like OncoEMR's `GenerateMergeAndCachePdfRS` that previously failed with "Arithmetic operation resulted in an overflow" because the server received no form data.

## Bug Fixes

- **browser**: Preserve POST body and content type when a form-submit popup with `target="_blank"` is converted into a BOSS tab — `BrowserHandleImpl` installs an engine-wide `BeforeSendUploadDataCallback` that records upload data from tracked popups, races URL detection against POST capture (500 ms grace), and replays the body via `LoadUrlParams.uploadData(...)` on the new tab's initial `loadUrl`; `BrowserServiceImpl.stashPopupPost` holds the captured `(url, body, contentType)` in a 10 s-TTL map that `createBrowser` consumes on first match. Fixes "print / export / open-in-new-window" flows built on form POST + `target="_blank"`. (#706)

## Improvements

- **plugin-api-browser**: Mirror the new POST-preservation surface in the plugin API module — `BrowserConfig` gains `initialPostData` / `initialPostContentType`, `BrowserHandle` adds `PopupNavigation` and `setOpenInNewTabWithDataCallback` (with default), and `BrowserService` gains a default `stashPopupPost` — so plugins (Fluck browser) compile against the updated contract. (#706)
