# BOSS v8.12.9

**Release Date**: October 2024

## Summary

This release introduces the Secret Manager admin plugin for secure credential management with server-side encryption, 2FA support, and multiple critical fixes for database encryption and RBAC functionality.

## Highlights

- New Secret Manager admin plugin with server-side encryption using PostgreSQL pgcrypto
- Fixed critical encryption function access issues in RPC functions with fully qualified schema references
- Corrected RBAC function return format resolving deserialization errors in role management
- Enhanced permission creation dialog with proper error handling and async operation feedback

## Features

### Secret Manager Admin Plugin
- **Server-side encrypted credential storage** (#87)
  - Three-table schema: secrets, secret_metadata, secret_tags
  - AES encryption using PostgreSQL pgcrypto extension
  - RLS policies ensuring users only access their own secrets
  - Unique constraint on (user_id, website, username) combinations

- **Comprehensive credential management UI** (#87)
  - Password visibility toggle (masked by default)
  - Collapsible 2FA metadata section with recovery codes
  - Tags displayed as colored badges for categorization
  - Expiration date tracking with warnings
  - Edit/Delete actions per secret

- **Search and pagination** (#87)
  - Search across website and username fields
  - Instagram-style infinite scroll with "load more" functionality
  - Type-safe data classes with validation

- **RBAC integration** (#87)
  - Admin-only access control
  - New 'secrets.write' permission for access control

## Bug Fixes

### Database & Encryption
- **Fixed PostgreSQL function resolution in SECURITY DEFINER functions** (#87)
  - Added fully qualified schema references (public.decrypt_text, public.encrypt_text)
  - Updated RPC functions: get_user_secrets(), search_user_secrets(), create_secret(), update_secret()
  - Functions can now access pgcrypto with SET search_path = '' security restriction
  - Fixed encryption round-trip: properly encrypts and decrypts credentials

- **Corrected encryption function schema references** (#87)
  - Added search_path = 'public, pg_catalog, extensions' to encryption functions
  - Explicitly qualified extension calls: extensions.encrypt(), extensions.decrypt()
  - Resolved "function decrypt_text(text) does not exist" errors

### RBAC System
- **Fixed RBAC function return format** (#87)
  - Changed get_all_roles() to return {"success": true, "data": [...]}
  - Changed get_all_permissions() to return {"success": true, "data": [...]}
  - Previously returned old format: {"roles": [...]} causing deserialization errors
  - Role Creation panel and Admin User Management now load without errors

### User Interface
- **Permission creation dialog improvements** (#87)
  - Dialog now stays open during async operations
  - Displays error messages (e.g., "Permission already exists")
  - Auto-closes only on successful creation
  - Shows loading indicator during operation
  - Disables form fields while in progress
  - Users can see errors and retry with different name

## Improvements

### Data Models
- **Enhanced serialization support** (#87)
  - Added secret_id field to RpcResponse for proper serialization
  - Type-safe data classes with kotlinx serialization

### Debugging & Logging
- **Improved error visibility** (#87)
  - Enhanced debug logging in SecretService
  - Detailed logging in createSecret() and getUserSecrets() methods

### Database Migrations
- **Migration organization improvements** (#87)
  - Renamed migration files to proper timestamp format
  - Fixed migration history sync issues
  - Added diagnostic migrations for debugging
