# BOSS v8.17.8

**Release Date:** 2026-03-05

Codebase cleanup release removing ~8,000 lines of dead Fluck browser UI code and consolidating browser infrastructure into the `plugin/browser` package now that the dynamic fluck-browser plugin fully replaces the internal browser UI.

## Highlights

- **Dead Fluck UI Removed**: Eight dead UI files (~8k lines) deleted from `tab_types/fluck/` â€” `JxBrowserCompose.kt`, `DownloadPanel.kt`, `SecretSelectionDialog.kt`, `SecretContextMenuBuilder.kt`, `QuickCreateSecretDialog.kt`, `SwingContextMenu.kt`, `SwingTextContextMenu.kt`, and `BrowserContextMenuCallback.kt`
- **Browser Infrastructure Relocated**: Eleven infrastructure files moved from `tab_types/fluck/` to `plugin/browser/` where `BrowserServiceImpl` and `BrowserHandleImpl` already live, collocating all browser logic in one package
- **BrowserSettingsConfig Extracted**: Browser settings configuration extracted into a dedicated `BrowserSettingsConfig.kt` in `plugin/browser/`

## Improvements

- **browser**: Delete 8 dead internal browser UI files (~8k lines) superseded by the fluck-browser plugin (v1.0.17)
- **browser**: Relocate 11 infrastructure files (`BrowserJavaScripts`, `BrowserSettingsManager`, `BrowserZoomSettingsManager`, `FluckEngine`, `FormFieldDetector`, `FormFieldInjector`, `LockedBrowser`, `ScreenCaptureNotifier`, `ScreenCapturePickerDialog`, `UrlHistoryManager`) from `tab_types/fluck/` to `plugin/browser/`
- **browser**: Extract `BrowserSettingsConfig` into its own file in `plugin/browser/`
- **browser**: Retain KMP `expect`/`actual` bridge in `tab_types/fluck/` as required by the Kotlin Multiplatform compilation model
