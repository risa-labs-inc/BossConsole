# BOSS v8.12.7

**Release Date**: October 2024

## Summary

This release introduces a comprehensive admin panel for dynamic role and permission creation, enabling administrators to manage access control at runtime without code changes. The system includes full CRUD operations, robust validation, and a complete migration from ENUM-based to table-based RBAC architecture.

## Highlights

- Admin role and permission creation panel with intuitive UI for runtime access control management
- Complete RBAC migration from PostgreSQL ENUMs to flexible table-based schema with delete support
- 17 comprehensive RPC functions for role and permission lifecycle management
- System role protection preventing accidental deletion of built-in roles

## Features

### Dynamic Role & Permission Creation Panel (#69, #76)

- **Admin-only panel** - Dedicated UI for creating and managing roles and permissions dynamically
- **Role selection dropdown** - View all roles (system + custom) with real-time permissions display
- **Create role dialog** - Name validation with regex patterns, description field, instant feedback
- **Create permission dialog** - Enforces `domain.action` format (e.g., "code.review") with validation
- **Permission assignment** - Checkbox-based interface for assigning/revoking permissions per role
- **Delete operations** - Remove custom roles/permissions with confirmation dialogs
- **System role badges** - Visual indicators for protected system roles (user, admin)

### Database Architecture Overhaul

- **Table-based RBAC schema** - Migrated from ENUMs to `roles` and `permissions` tables with UUID primary keys
- **Rich metadata support** - Added `id`, `description`, `is_system`, and timestamp fields
- **17 RPC functions** - Complete API for all CRUD operations with CASCADE delete support
- **System role protection** - `is_system` flag prevents deletion of built-in roles
- **Foreign key relationships** - `user_roles` and `role_permissions` use UUID foreign keys
- **Auth hook integration** - Updated `custom_access_token_hook` to query new table structure

### RPC Functions

- `create_new_role()` - Create roles dynamically with validation
- `create_new_permission()` - Create permissions with `domain.action` format
- `get_all_roles()` / `get_all_permissions()` - Query all roles/permissions with metadata
- `get_role_permissions()` - Get permissions assigned to a specific role
- `assign_permission_to_role()` / `remove_permission_from_role()` - Manage role-permission mappings
- `delete_role()` / `delete_permission()` - Delete custom roles/permissions with system role validation
- All functions require admin role with `SECURITY DEFINER` and regex validation

### Service Layer

- **RoleCreationService.kt** - Complete Kotlin API for role/permission operations
- **Client-side validation** - `validateRoleName()` and `validatePermissionName()` functions
- **Result types** - Type-safe error handling with Kotlin Result
- **Supabase integration** - Postgrest RPC client for database operations
- **Helper functions** - Backward compatibility support in RoleService.kt

### UI Components

- **RoleCreationViewModel.kt** - Lifecycle-aware state management with coroutine support
- **RoleCreationPanel.kt** - Plugin registration for admin-only panel
- **RoleCreationView.kt** - Main UI with vertical stacked layout optimized for narrow panels
- **RoleCreationDialogs.kt** - Modal dialogs for create/delete operations
- **Material Design** - Compose Multiplatform UI with consistent styling

### Validation Rules

**Role names:**
- Pattern: `^[a-z][a-z0-9_]{2,50}$`
- 3-50 characters, lowercase letters, numbers, and underscores only
- Must start with a letter
- Reserved names blocked: user, admin, authenticated, anon

**Permission names:**
- Pattern: `^[a-z][a-z0-9_]{1,30}\.[a-z][a-z0-9_]{1,30}$`
- domain.action format (e.g., "code.review", "ai.train")
- Each part 2-30 characters
- Lowercase letters, numbers, and underscores only

## Bug Fixes

### Passkey Authentication (#77)

- **Fixed admin role injection** - Passkey login now correctly includes admin role in JWT claims
- Updated `getUserRoles()` in jwt.ts to JOIN with roles table instead of querying old ENUM schema
- Changed from `.select('role')` to `.select('roles!inner(name)')` with nested structure extraction

### RBAC System

- **Fixed RPC function serialization** (#76)
  - Force-dropped old ENUM-based functions with `DROP IF EXISTS CASCADE`
  - Recreated with proper table-based schema returning `{success, data: [...]}` format
  - Resolved "unknown key" errors where Kotlin expected `data` but received `roles`

- **Fixed delete operations** (#70)
  - Implemented `delete_role()` and `delete_permission()` functions
  - Added system role validation to prevent deletion of built-in roles
  - CASCADE deletes properly clean up `user_roles` and `role_permissions` references

- **Fixed null safety issues** (#76)
  - Replaced dangerous `!!` operators with `.getOrThrow()`
  - Migrated to `.onSuccess/.onFailure` pattern for Result handling
  - Fixed in RoleCreationViewModel.kt, AdminRoleManagementViewModel.kt, UserService.kt
  - Prevents potential null pointer crashes

## Breaking Changes

### RBAC Architecture

This release completes the migration from ENUM-based to table-based RBAC. The underlying database schema has changed significantly, but the client API remains compatible for most use cases.

**Database changes:**
- `app_role` and `app_permission` ENUMs replaced with `roles` and `permissions` tables
- `user_roles.role` and `role_permissions.permission` changed to UUID foreign keys
- Legacy backup tables removed: `user_roles_old`, `role_permissions_old`

**RPC function updates:**
- All functions now return table-based data structures with rich metadata
- Response format: `{success: boolean, data: [...]}` or `{success: boolean, role: ..., permissions: [...]}`

**Auth hook required:**
- Custom access token hook must be manually enabled in Supabase Dashboard
- Path: Authentication → Hooks → Custom Access Token
- Required for JWT tokens to include RBAC claims
