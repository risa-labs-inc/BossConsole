# BOSS v9.2.45

**Release Date:** 2026-07-12

Audits the JxBrowser engine flags for Chromium 150, pre-warms the engine in the background so cold-start no longer freezes the UI, and stops the first browser tab from stalling on the Widevine CDM download.

## Highlights

- **Chromium 150 flag audit**: Per-platform, OFF_SCREEN-aware switch cleanup — drops flags that are now default-on or blocklist-forcing (`--enable-gpu-rasterization`, `--enable-zero-copy`, `--ignore-gpu-blocklist`), restores the sandbox (`BOSS_CHROMIUM_DISABLE_SANDBOX` escape hatch), scopes `--disable-dev-shm-usage` to containers only, and adds targeted per-OS switches (Windows occlusion, Linux VA-API decode). The flag decision is extracted as a pure `performanceSwitchesFor(...)` with per-platform unit tests, plus `BOSS_CHROMIUM_EXTRA_SWITCHES` for no-rebuild A/B testing (whitespace-separated, logged for audit) (#864).
- **Background engine pre-warm**: `FluckEngine.prewarmInBackground()` boots the engine off the UI thread from `main.kt` instead of synchronously inside the first browser tab's composition — eliminating a multi-second cold-start freeze (~1–1.3s boot, measured). Gated on prior browser use so terminal-/editor-only sessions and first runs never pay the Chromium spawn; opt out with `BOSS_BROWSER_PREWARM=false`.
- **Async Widevine**: The Widevine CDM activation is no longer `join()`ed during engine creation (the network download stalled the first tab), and `BrowserService.isAvailable()` no longer triggers engine init from the UI thread — init happens lazily in `createBrowser`.

## Improvements

- **browser**: audit JxBrowser engine flags for Chromium 150, pre-warm the engine on a background thread, and make Widevine activation async. The flag audit is per-platform and OFF_SCREEN-aware (drops default-on/blocklist-forcing GPU flags, sandbox back ON with `BOSS_CHROMIUM_DISABLE_SANDBOX`, container-only `--disable-dev-shm-usage`, Windows `CalculateNativeWinOcclusion` off, Linux VA-API decode, 512MB disk cache, `BOSS_CHROMIUM_EXTRA_SWITCHES` for no-rebuild A/B). Pre-warm is gated on the browser profile already existing and self-heals a background boot failure so it can never make `isAvailable()` report false, with `@Volatile` init-state fields and identity-checked cleanup guarding the UI-thread/pre-warm-thread interleavings. Container detection reads `/.dockerenv` and `/proc/1/cgroup` (kubepods/containerd/lxc/podman) with a `BOSS_IN_CONTAINER` override and uses the supported `EngineOptions.disableSandbox()` API. SkiaGraphite was A/B-tested live on Apple Silicon and reverted to opt-in (`BOSS_ENABLE_SKIA_GRAPHITE=true`) because it blanks the OSR frame output on JxBrowser 9.3.0. Linux sandbox/zygote/user-namespace bring-up failures now carry an actionable hint. Adds `FluckEngineSwitchesTest` covering the per-platform decision, extra-switch partitioning, and the cgroup container predicate (#864).
