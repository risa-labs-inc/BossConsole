# BOSS v8.16.29

**Release Date:** 2026-02-08

Maintenance release adding screen capture support for browser plugins with enhanced permission APIs.

## Highlights

- **Screen Capture API**: New ScreenCaptureProvider interface enables external browser plugins to access screen/window/tab sharing
- **Browser Plugin Support**: Proper screen share dialog integration for JxBrowser capture requests
- **macOS Camera Permissions**: Added camera permission checks for macOS platform

## Features

- **plugin-api**: Add ScreenCaptureProvider and fix screen share dialog (#614)
  - Add ScreenCaptureProvider interface to plugin-api for external plugins to access screen capture permission APIs and internal browser tabs
  - Implement DesktopScreenCaptureProvider in host app with proper lifecycle management
  - Wire up ScreenCapturePickerDialog in BossWindow.kt to observe and display capture requests from JxBrowser's StartCaptureSessionCallback
  - Add debug logging for VIDEO_CAPTURE and AUDIO_CAPTURE permission requests
  - Add bounds checking for source index in selectSource to prevent IndexOutOfBoundsException
  - Bump plugin-api version to 1.0.13
  - Fixes screen share functionality for external browser plugins (like fluck-browser)

## Improvements

- **platform**: Add macOS camera permission check with NSCameraUsageDescription support
- **screen-capture**: Enhanced error handling with bounds checking before accessing capture source arrays
