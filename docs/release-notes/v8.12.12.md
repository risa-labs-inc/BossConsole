# BOSS v8.12.12

**Release Date**: October 2024

## Summary

This release introduces comprehensive secret management capabilities with user-level secret viewing, role-based secret sharing with full audit trails, and database infrastructure improvements.

## Highlights

- New "My Secrets" panel for viewing owned and shared secrets
- Secret sharing with users and roles with view-only access control
- Safe-by-design sharing with searchable user/role dropdowns
- Complete audit logging for secret access tracking
- Database migration consolidation and development workflow improvements

## Features

### Secret Management
- **User-level secret list plugin** (#97, Issue #80)
  - Read-only "My Secrets" panel for authenticated users
  - View website:username credentials for owned and shared secrets
  - Ownership badges showing "Owner" or "Shared" status
  - One-click username copy to clipboard
  - Client-side search and filtering
  - Expandable metadata cards (tags, notes, expiration, 2FA info)
  - Pagination support for large secret lists
  - Dynamic panel registration based on authentication state
  - Fixed `get_user_secrets_with_shared()` with correct AES+base64 decryption
  - Added `DISTINCT ON` to prevent duplicate entries
  - Fixed tags aggregation from `secret_tags` table
  - Added graceful error handling for corrupt encrypted data

- **Secret sharing with privilege management** (#96, Issue #81)
  - Share secrets with both individual users AND roles
  - View-only access level enforced via RLS policies
  - Admin override capability through RLS policies
  - Full audit trail logging via `secret_access_log` table
  - Real-time share list updates with revoke capability
  - Optional notes and expiration dates for shares
  - Searchable user dropdown with live email search (server-side)
  - Role dropdown with descriptions for safer selection
  - Safe-by-design: eliminates typo-based sharing errors
  - New `secret_shares` table for user/role targeting
  - Four database functions for share management operations

## Improvements

### Database
- **Migration consolidation and development guidelines** (#98)
  - Replaced 11 migration files (109KB) with 14 well-documented, logically organized migration files
  - Comprehensive documentation throughout all migrations
  - Organized by dependency: extensions → functions → tables → constraints → indexes → policies → grants
  - Fixed RLS policies for `completed_authentications` table (7 policies)
  - Total: 13 tables, 42 functions, 35 indexes, 50+ RLS policies, 150+ grants
  - Added migration best practices to CLAUDE.md
  - Established workflow: test locally before committing
  - Documented periodic consolidation process

### Configuration
- Enabled `custom_access_token_hook` for local development (JWT with RBAC claims)
- Fixed Edge Function JWT verification (`verify_jwt = false` for passkey function)
- Removed test user from seed data for production-safe configuration

## Bug Fixes

### Database
- Fixed `share_secret()` double INSERT bug
- Fixed `get_secret_shares()` PostgreSQL type mismatch (varchar to text casting)
- Fixed tags aggregation and duplicate prevention in secret queries
