# BOSS v9.1.10

**Release Date:** 2026-05-20

Auto-updates non-downloadOnly system plugins in the background so long-running installs no longer stay pinned to whatever version was latest at first launch, and unblocks the Compose Multiplatform 1.11.0 upgrade by replacing the now-ERROR-level deprecated `Key.Home` constant with `Key.MoveHome`.

## Highlights

- **Background auto-update for system plugins**: System plugins like `plugin-manager`, `terminal-tab`, `fluck-browser`, `editor-tab`, `terminal`, and `boss-plugin-api` now run the same `scheduleBackgroundUpdateCheck` path that the microkernel runtime already uses. Long-running installs that were stuck on an old `plugin-manager` (e.g. 1.4.32 vs. an actual latest of 1.4.41) will pick up newer JARs on next launch via the existing `.tmp` + atomic rename staging.
- **Preserves enabled/sourceUrl across auto-updates**: `addInstalledPlugin` does removeIf+add, so the auto-update path now reads the existing entry first and forwards `enabled` / `sourceUrl` — background updates can't silently re-enable a user-disabled plugin or wipe sourceUrl tracking.
- **Compose Multiplatform 1.11.0**: Bumps the UI toolkit to 1.11.0. `Key.Home` (intended for Android's system home key, not the keyboard Home key) was promoted from deprecation-warning to ERROR in 1.11.0 and was breaking `compileKotlinDesktop` — replaced with `Key.MoveHome` in `MenuShortcutBridge.kt`.
- **BossTerm 1.1.96**: Picks up the latest terminal plugin release.

## Features

- **plugins**: Auto-update non-downloadOnly system plugins in the background. `ensureSystemPluginsInstalled` now kicks off `scheduleBackgroundUpdateCheck` for any system-plugin entry already on disk; the existing download flow (`downloadSystemPluginFromGitHub`) handles staging via `.tmp` + atomic rename, old-version cleanup, and PluginPersistence registration. The current session keeps its already-loaded classloader (hot-swapping under live class references would break callers), so the new JAR becomes the one loaded on next launch. Two log strings and `scheduleBackgroundUpdateCheck`'s KDoc were also relaxed so they read sensibly for both downloadOnly and regular system plugins. (#710)

## Bug Fixes

- **keymap**: Replace deprecated `Key.Home` with `Key.MoveHome` in `MenuShortcutBridge.kt`. Compose Multiplatform 1.11.0 promoted the `Key.Home` deprecation to ERROR level — the constant is meant for Android's system home key, not the keyboard Home key — which was breaking `compileKotlinDesktop`.

## Improvements

- **plugins**: Auto-update review follow-ups — `addInstalledPlugin` now reads the existing entry and forwards `enabled` / `sourceUrl` so background updates don't silently re-enable user-disabled plugins or wipe sourceUrl tracking, and the three update operations were reordered so persistence never points to a JAR that hasn't landed yet. (#710)
