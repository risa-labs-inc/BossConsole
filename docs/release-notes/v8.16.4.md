# BOSS v8.16.4 Release Notes

**Release Date:** 2026-01-23

## Summary

This release resolves critical multi-window state management issues by removing global project and runner configuration state, ensuring each window operates independently.

## Highlights

- Removed global project selection state to fix multi-window conflicts
- Isolated run configuration state per window for independent operation
- Fixed Git panels to hide when no repository is selected
- Improved keyboard shortcut reliability with MenuBar as single source of truth

## Bug Fixes

### Multi-Window
- **Remove global project state and hide git panels when no repo selected** ([#526](https://github.com/risa-labs-inc/BossConsole/pull/526))
  - Git panels (Git Log and Git Changes) now dynamically hide when no project is selected or repository is not a Git repo
  - Follows SecretManagerPanel pattern for dynamic panel registration
  - Each window independently manages panel visibility based on its own project selection
  - Fixed keyboard shortcuts triggering multiple windows (Cmd+N) by removing redundant handlers
  - MenuBar now serves as single source of truth for all keyboard shortcuts at OS level
  - Removed GlobalKeyboardInterceptor and BossActionHandler classes for cleaner architecture

- **Remove global ProjectState.selectedProject API** ([#525](https://github.com/risa-labs-inc/BossConsole/pull/525))
  - Eliminated global project selection API that caused cross-window interference
  - All project selection now properly scoped to individual windows via WindowProjectState
  - Updated 19 files to use per-window state instead of global state
  - Removed global RunConfigurationManager.selectedConfiguration in favor of WindowRunnerState
  - Each window maintains independent run configuration selection
  - Added WindowRunnerStateRegistry cleanup to prevent memory leaks on window close
  - Fixed test flakiness in KeyboardEventBusTest with proper coroutine cancellation

## Improvements

### Architecture
- Window-scoped state management now consistent across project selection, run configurations, and panel visibility
- Cleaner keyboard shortcut architecture following BossTerm patterns
