# BOSS v8.17.10

**Release Date:** 2026-03-06

Plugin stability release introducing automatic detection and graceful handling of binary-incompatible plugins, crash recovery tooling, and a cleaner New Tab dialog that hides tabs for disabled plugins.

## Highlights

- **Auto-disable Incompatible Plugins**: Binary incompatibility errors (`NoSuchMethodError`, `NoClassDefFoundError`, `UnsatisfiedLinkError`) are caught at registration time and immediately disable the offending plugin without entering a crash/restart loop
- **Clean Data & Restart Recovery**: A new "Clean Data & Restart" button in the crash report dialog deletes `~/.boss` and relaunches the app, giving users a recovery path when corrupted plugins cause persistent crashes
- **New Tab Dialog Cleanup**: Disabled plugin tab types (URL, File, Terminal) are now filtered out of the New Tab dialog; an empty-state message is shown when no tab plugins are enabled (#640)
- **Uninstall Fix for Incompatible Plugins**: `DynamicPluginManager.uninstallPlugin()` now handles plugins already evicted from the classloader during incompatibility handling, allowing the Plugin Manager update flow to proceed normally (#640)

## Bug Fixes

- **plugins**: Catch `Throwable` (not just `Exception`) in `installPlugin`, `enablePlugin`, `loadPersistedPlugins`, and `loadBundledPlugins` so `LinkageError` subclasses no longer escape to `CrashHandler`
- **plugins**: Fix uninstall of already-unloaded incompatible plugins to unblock the reinstall/update flow (#640)
- **tabs**: Filter New Tab dialog tab types against `TabRegistry` so tabs for unregistered plugins are hidden (#640)

## Features

- **plugins**: `PluginErrorClassifier` classifies errors as `BINARY_INCOMPATIBILITY` or `RECOVERABLE` by walking the cause chain
- **plugins**: Incompatible plugins are marked `DISABLED` with an `isIncompatible` flag and remain visible in the Plugin Manager instead of being silently dropped
- **plugins**: `PluginErrorFallback` shows a "Plugin Update Required" card with a `SystemUpdate` icon for incompatible errors; `PluginDisabledFallback` shows a "Check for Updates" button variant
- **crash**: "Clean Data & Restart" action in `CrashReportDialog` clears `~/.boss` (or `~/.boss_debug`) and restarts the application
