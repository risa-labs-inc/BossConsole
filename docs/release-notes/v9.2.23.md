# BOSS v9.2.23

**Release Date:** 2026-07-03

BossEditor moves out of the host and into the `editor-tab` plugin — the same arrangement as BossTerm inside `terminal-tab`. The host sheds the whole PSI/compiler stack (~60 MB off the distributable), the plugin now owns editor rendering plus editor/LSP settings, and a new system-plugin `minVersion` gate makes sure an up-to-date editor plugin is present before the first editor tab opens.

## Highlights

- **BossEditor is no longer a host dependency**: `composeApp` drops `bosseditor-compose-desktop` and `kotlin-compiler-embeddable` (~60 MB off the host distributable) — BossEditor now ships inside the `editor-tab` plugin JAR (>=1.4.0), which bundles the full PSI stack (#818).
- **Editor/LSP settings served by the plugin**: New `EditorTabPluginAPI` (mirroring `TerminalTabPluginAPI`) lets the plugin provide the editor and LSP settings panels; host settings sections delegate through `EditorAPIAccess` instead of calling BossEditor internals directly, and self-heal to the real panel once the plugin's async API registration lands (#818).
- **`minVersion` gate for system plugins**: An installed `editor-tab` older than the host requires is force-updated synchronously before load, so the first launch of this host can't `NoClassDefFoundError` on every editor tab; offline hosts keep the old JAR and retry next launch (#818).

## Features

- **plugin-api**: `EditorTabPluginAPI` with default no-op `EditorSettingsPanel` / `LspSettingsPanel` composables; host settings sections delegate through `EditorAPIAccess`, and the `editor-tab` plugin implements the panels against its bundled BossEditor (#818)
- **plugins**: `SystemPluginInfo.minVersion` forces a synchronous update before load when the installed JAR predates the host's minimum, with an offline fallback that retries next launch (#818)
- **settings**: When the editor plugin is absent, still loading, or predates `EditorTabPluginAPI`, settings sections show a short muted explanation instead of a blank panel that reads as a bug (#818)

## Bug Fixes

- **plugins**: Never install a `-thin.jar` release asset — plugin releases can carry the module's default `:jar` output next to the real plugin JAR (and GitHub may list it first), so asset selection now prefers the first non-thin match instead of a jar missing everything `buildPluginJar` bundles (#818)
- **plugins**: Verify the *downloaded* release satisfies `minVersion`, not just the trigger — if the repo's latest release still predates the host's minimum (release-order violation, yanked release), keep the on-disk JAR and retry next launch rather than persisting a contract-breaking JAR (#818)
- **plugins**: `isNewerVersion` parses segments by numeric prefix (`0-rc1` → 0) and, on a numeric tie, a release beats its own pre-release, so `1.4.0-rc1` no longer satisfies `minVersion = "1.4.0"` (#818)

## Improvements

- **editor**: Delete the dead host-side BossEditor renderer — `BossEditorIntegration`, `DesktopCodeEditorUI`, `FileModificationTracker`, and `EditorSearchEventBus` backed only unused content paths; desktop's `PlatformCodeEditorUI` and `CodeEditorContent` now delegate to the shared basic `CodeEditorUI` like the other platforms (#818)
- **editor**: `EditorContentProviderImpl` is reduced to what the host still owns — file I/O, language detection, file-open routing, run configs, font enumeration — with editor-internal methods falling back to their plugin-api defaults; `SemanticTokenProviderImpl` and `NavigationResolverProviderImpl` are deleted (the plugin runs its own semantic provider; no navigation-resolver consumer existed) (#818)
- **startup**: `main.kt` loses the PSI warm-up and shutdown-hook calls now that the plugin owns that lifecycle; `EditorContentProviderImpl` logs (rather than swallows) main-function failures and documents that the fallback editor is view-only (#818)
- **settings**: `DefaultPlugin` exposes an `apiRegistryVersion` `StateFlow` bumped on `registerPluginAPI`; `EditorAPIAccess.rememberProvider()` observes it so the "plugin isn't loaded yet" notice swaps to the real panel when registration lands, and the double `getProvider()` lookup per composition is removed (#818)
- **tests**: `PluginStoreSetupMinVersionGateTest` covers the load/update decision, comparator edges, thin-jar asset skipping, and filename version extraction (mirrors `PluginStoreSetupIpcGateTest`) (#818)

## Dependencies

- **editor**: Requires `editor-tab` >= 1.4.0 for editor/LSP settings panels and semantic highlighting (#818)
