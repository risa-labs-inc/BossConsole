# BOSS v9.2.12

**Release Date:** 2026-06-20

Adds BossTerm magic-link sign-in to the shared auth backend, a host JS↔Java bridge for co-browse tab sharing, and host system events on the application event bus, alongside fixes for the Windows branded-Chromium release build, terminal launches in paths with spaces, and the update dialog layout.

## Highlights

- **BossTerm magic-link sign-in** (#789): The auth backend now supports BossTerm against the shared Supabase user pool, branching entirely on a `redirect_to == bossterm://auth/verify` signal — per-app email/page branding, a `bossterm://auth/verify` deeplink, and verbatim `type` passthrough so a new user's first link verifies as `signup`. Also closes a pre-existing reflected-XSS hole in the redirect page by output-escaping the deeplink per context.
- **Co-browse tab sharing bridge** (#786): A host JS↔Java bridge lets shared browser tabs communicate with the host application.
- **Host system events on the event bus** (#788): Analytics now emits host system events onto the application event bus so they can be observed by other subsystems.

## Features

- **auth**: BossTerm magic-link sign-in on the shared auth backend — per-app email/page branding, `bossterm://auth/verify` deeplink, and `type` passthrough (`signup` survives for first-time users), with the redirect function refactored into an edge-runtime-safe entrypoint (`app.ts` + thin `index.ts`) and a deno test suite. (#789)
- **browser**: Host JS↔Java bridge enabling co-browse tab sharing. (#786)

## Bug Fixes

- **ci**: Fix the branded-Chromium `windows-x64` build hang and add an incomplete-release guard. (#790)
- **terminal**: Shell-quote `{projectPath}` when launching terminals so folders with spaces in their path open correctly. (#780)
- **panel**: Stop the update dialog from growing vertically while scrolling through release notes. (#778)
- **auth**: Output-escape the redirect-page deeplink per context (JSON-stringified in the inline script, HTML-attribute-escaped in the href), neutralizing a pre-existing reflected-XSS vector via crafted token params. (#789)

## Improvements

- **analytics**: Emit host system events onto the application event bus. (#788)
