# BOSS v8.17.15

**Release Date:** 2026-03-07

Database performance and security hardening release adding optimized Postgres views for user/role and plugin store queries, with critical RLS fixes ensuring row-level security policies are properly enforced for the calling role.

## Highlights

- **`users_with_roles` view**: Replaces N+1 RPC calls with a single `SELECT` joining users, user\_roles, and roles tables with array aggregation
- **`plugins_with_latest_version` view**: Eliminates N+1 pattern in plugin store listing via a `LATERAL JOIN` that fetches the latest version per plugin in one query
- **RLS bypass fix**: Both views now use `security_invoker = true` so RLS policies on underlying tables are enforced for the calling role, not the superuser view owner
- **Anonymous access revoked**: `anon` role can no longer `SELECT` from `users_with_roles`

## Features

- **database**: Add `users_with_roles` Postgres view — joins `users`, `user_roles`, and `roles` with array aggregation for efficient single-query user+role lookups
- **database**: Add `plugins_with_latest_version` Postgres view — `LATERAL JOIN` view for efficient latest-version-per-plugin resolution, eliminating N+1 in plugin store listing

## Security

- **database**: Fix RLS bypass on `users_with_roles` and `plugins_with_latest_version` views by setting `security_invoker = true`, ensuring RLS policies on underlying tables are applied for the calling role
- **database**: Revoke `SELECT` on `users_with_roles` from the `anon` role; add explicit grants on `plugins_with_latest_version`
- **database**: Add covering index on `plugin_versions` for deterministic latest-version ordering (tiebreaker: `pv.id DESC`)

## Improvements

- **database**: Remove redundant migration `000003` after repairing remote migration history via `supabase migration repair`
