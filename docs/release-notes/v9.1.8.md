# BOSS v9.1.8

**Release Date:** 2026-05-14

Adds a Customize Sidebar menu (per-slot hide/show submenus and a draggable three-dot button that relocates between left/right sidebar slots, with a matching View → Customize Sidebar… entry), and fixes a main-panel terminal regression where the cursor blinked but keystrokes were dropped after focus round-tripped through another panel.

## Highlights

- **Customize Sidebar menu**: New three-dot button at the bottom of the sidebar opens a cascading per-slot submenu listing every registered panel (left + right) with a visibility checkmark. Choices persist to `~/.boss/sidebar-visibility-settings.json` (allow-by-default so newly installed plugins appear without opt-in).
- **Draggable Customize button**: The customize button itself can be long-pressed and dragged to any of the four sidebar slots (`left.top.top`, `left.top.bottom`, `left.bottom`, `right.top.top`, `right.top.bottom`) — drops outside a valid slot snap back. Lives outside `itemsBySlot` so its synthetic id never leaks into plugin slot state.
- **View → Customize Sidebar…**: OS menu-bar entry that opens the same popup anchored to wherever the button currently lives. In focus mode, the trigger force-reveals the hosting sidebar via a per-window keyed `StateFlow<Map<String, Long>>` so cross-window requests can't clobber each other.
- **Main-panel terminal focus fix**: `LocalIsPanelActive` composition local now flows the host's active-panel signal into BossTerm's public `TabbedTerminal`, so the per-pane `FocusRequester` re-fires whenever focus returns to the terminal from a browser split or sidebar plugin.

## Features

- **sidebar**: Customize Sidebar — three-dot button with per-slot cascading submenus listing every registered left + right panel, drag-and-drop relocation between any of the five sidebar slots, View → Customize Sidebar… menu entry, persisted to `~/.boss/sidebar-visibility-settings.json`. Hidden panels also have their open content dismissed via `BossDraggableComponent.closePanelForItem`. (#705)

## Bug Fixes

- **terminal**: Fix main-panel terminal losing keyboard focus after clicking out to another panel and back. The public `TabbedTerminal` composable previously hardcoded `isActiveTab = true`, so BossTerm's `LaunchedEffect(tab.id, isActiveTab)` refocus path never re-fired across host focus round-trips. Introduced `LocalIsPanelActive: ProvidableCompositionLocal<Boolean>` in `plugin-api-core` so panel plugins can read the host's active-panel signal without coupling to internal split-view state; `BossMainWindowPanel` provides it from `splitViewState.setActivePanel`. (#709)

## Improvements

- **sidebar**: Customize-menu review follow-ups — switched `setHidden` / `setCustomizeButtonSlot` to `MutableStateFlow.update { }` so concurrent toggles can't clobber each other, replaced the implicit `.value` read in `getItemsForSlot` with an explicit hidden-set parameter (no longer relies on a sacrificial `collectAsState` to wire snapshot observation), reserved the `__` id prefix for synthetic sentinels, and routed left/right slot detection through `SidebarVisibilitySettings.isLeftSide`. (#705)
- **drag-and-drop**: New `BossDraggableComponent.cancelDragSnapBack()` mirrors `stopDragging()`'s bookkeeping reset without running its slot-mutation tail, so a mid-drag `DisposableEffect` cleanup on the synthetic customize button can't inject its id into `itemsBySlot[dropTargetSlot]`. (#705)
- **context-menu**: `ContextMenu` gained an `alignment` parameter and now renders `trailingIcon` inside `SubMenuContent` so submenu items can show checkmarks; the customize popup uses `Alignment.BottomStart` / `BottomEnd` with a sidebar-width `IntOffset` so it lands flush against the sidebar's outer edge instead of overlapping the icon strip. (#705)

## Dependencies

- **deps**: Bump `com.risaboss:bossterm-compose` from `1.0.93` to `1.0.94` — adds the `isActive` parameter on the public `TabbedTerminal` composable that the terminal-focus fix depends on. (#709)
