# BOSS v9.2.17

**Release Date:** 2026-06-29

Adds keyboard tab switching (Ctrl+Tab / Ctrl+Shift+Tab) with positional and Alt+Tab-style MRU modes plus a VS Code-style switcher overlay, and gates plugin-store installs on a plugin's `requiredPermissions` so a user can't install a plugin they wouldn't be allowed to use.

## Highlights

- **Ctrl+Tab tab switching**: Next/previous tab switching within the active panel, wired through the existing keymap pipeline. Two cycling modes selectable in Settings > Shortcuts — Positional (next/previous in tab-bar order, wrapping at the ends) and MRU (Alt+Tab style, committing the landed tab on modifier release) — with an MRU switcher HUD overlay that reuses the app's design tokens (#805).
- **Install gated on permissions**: Installing a plugin from the store now requires the caller to hold the plugin's `requiredPermissions` — the same list the host already uses to gate a plugin's visibility/use after install — making it a single source of truth. An empty list is the `user.read` baseline, so legacy plugins stay installable by everyone (#806).

## Features

- **tabs**: Add Ctrl+Tab / Ctrl+Shift+Tab to switch between tabs in the active panel, wired through the keymap pipeline (KeymapActions → presets → AWTKeyboardInterceptor → MenuActionsHandler → BossApp → BossTabsComponent). Positional (default) and MRU (Alt+Tab style) cycling modes are selectable in Settings > Shortcuts; the MRU mode shows a centered, theme-aligned switcher HUD overlay while cycling. Bindings are added to all keymap presets so `migrateSettings()` merges them into existing users' keymaps on launch, and the new `tabSwitchMode` field on `KeymapSettings` is defaulted so old keymap files stay compatible. Includes a serialization round-trip / backward-compat test (#805)
- **plugin**: Gate plugin-store installs on `requiredPermissions` — the `/download` route returns 403 when a non-admin lacks the permission(s) (empty stays open), the publish path persists `requiredPermissions` on create and both GitHub paths (version-publish refreshes the gate to the latest manifest), and browse/detail/download responses expose it. Backed by a migration adding `plugins.required_permissions text[] NOT NULL DEFAULT '{}'` and recreating `get_plugin_with_stats` / `search_plugins` to surface it, with pgTAP coverage for the column shape and RPC surfacing. Backward compatible: existing plugins default to `{}` (open) (#806)

## Dependencies

- **deps**: Bump `com.google.protobuf:protoc` from 4.35.0 to 4.35.1 (#804)
- **deps**: Combined Dependabot bumps — gRPC, `org.graalvm.buildtools.native` (1.1.0 → 1.1.3), and `com.vanniktech.maven.publish` (0.36.0 → 0.37.0) (#807)
