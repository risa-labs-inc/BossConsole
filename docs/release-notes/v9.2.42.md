# BOSS v9.2.42

**Release Date:** 2026-07-11

Caps the number of plugin icons shown per sidebar slot behind a "More" overflow menu so a crowded rail never clips, and fixes two browser/plugin regressions: pinch-zoom firing on hidden or non-hovered browser views, and open sidebar panels rendering stale code after a plugin hot reload.

## Highlights

- **Sidebar plugin icons are capped per slot with a "More" overflow menu**: each icon slot now shows a limited number of plugin icons and collapses the rest behind a vertical three-dot "More" button (whose menu toggles panels exactly like the icons). A new **Settings → Sidebar** setting picks the cap — **ADAPTIVE** (default: each bar splits its measured height into a 40dp icon-row budget across slots round-robin, so no slot starves its neighbours and every plugin stays reachable at any window size) or **FIXED** (1..20 icons per slot). A right-click "Sidebar settings" entry on every plugin icon and the More button opens the new Sidebar section directly (#860).
- **Trackpad pinch no longer zooms hidden or non-hovered browser tabs**: the macOS magnification listener was attached window-wide, fired unconditionally, and leaked a new permanent listener on every browser view mount — so a pinch anywhere zoomed the fluck browser even behind another tab or in a different split. Pinch now only zooms when the cursor is actually over that `BrowserView`, the listener is registered via a `DisposableEffect` (removed on hide/close/dispose), and same-domain tabs zoom independently via `ZoomMode.PER_BROWSER` (#857).
- **Open sidebar panels refresh on plugin hot reload**: an open panel kept its cached component, so a hot reload that swapped the registry factories (unload+load, Toolbox update, disable→enable) kept rendering the *old* build and pinned the pre-reload classloader. Panels now reset to the new build once it registers — the panel counterpart of the tab-teardown on the other side of the API swap (#858).

## Bug Fixes

- **browser**: gate pinch-zoom on cursor position and view lifecycle — `BrowserHandleImpl` tracks pointer Enter/Exit over its `BrowserView` and only zooms when the cursor is over that view and the handle is still valid; registration moves to a `DisposableEffect` so the listener is removed when the view leaves composition; `MacOSGestureHandler.addMagnificationListener` returns a removable registration token and keeps its smoothing accumulator per listener instead of a shared global. Also sets `ZoomMode.PER_BROWSER` at handle construction so same-domain tabs zoom independently (Chromium's default `PER_ORIGIN` propagated a zoom to every tab on the same URL), fixes gesture-listener removal (the proxy's invocation handler returned `false` for every `equals`, so removal silently leaked — the proxy now implements identity `equals`/`hashCode`), binds the view and listener to the actual hosting window via `LocalWindowId` + `WindowFocusManager` (rebinding when a tab moves across windows) instead of a "first showing window" heuristic, and logs suppressed pinch zooms for field diagnosis (#857).
- **plugins**: refresh open sidebar panels on plugin hot reload — a new global `PanelComponentStoreRegistry` (windowId→store, mirroring `SplitViewStateRegistry`, tied to composition teardown) plus a `DynamicPluginManager.pluginPanelsRefresh` hook (invoked after `installPlugin`/`enablePlugin` release their mutex, passing the plugin's freshly-registered panel ids from the manager's own tracker) drive `PluginLoaderDelegateImpl.refreshPluginPanels`, which resets every matching open slot in every window on the EDT via the existing `resetComponent` primitive. `resetComponent` is hardened to isolate the old component's `onBeforeReset` (a closed classloader throws `Error`s, not `Exception`s) and always drop the stale instance; a redundant `enablePlugin` skips the refresh so it can't discard an open panel's in-memory state (#858, fixes #856).

## Features

- **sidebar**: limit plugin icons per slot with a "More" overflow menu, capped ADAPTIVE (height-derived 40dp row budget split round-robin, overflow slots spending their last row on the More button) or FIXED (1..20), persisted in `sidebar-visibility-settings.json`; adds a right-click "Sidebar settings" context-menu entry on plugin icons and the More button, and deduplicates the sidebar item click idiom into `BossDraggableComponent.handleSidebarItemClick` shared by the rail icon and the More menu (#860).
