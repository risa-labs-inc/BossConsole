# BOSS v8.16.42

**Release Date:** 2026-02-12

Critical stability fix preventing macOS crash when closing the application while in fullscreen mode.

## Highlights

- **macOS Fullscreen Fix**: Eliminated SIGABRT crash when closing app in fullscreen mode by properly exiting fullscreen before browser disposal

## Bug Fixes

- **browser**: Prevent SIGABRT crash when closing app in fullscreen (#626)
  - Exit fullscreen and maximized state BEFORE disposing JxBrowser instances during window close
  - The crash occurred because macOS runs an async fullscreen exit transition, and JxBrowser's AwtHelperLibrary tries to access the AWT window handle during this transition when the window is in a half-disposed state
  - Reset Frame.MAXIMIZED_BOTH to NORMAL before browser disposal
  - Request native macOS fullscreen exit via com.apple.eawt.Application
  - Use GraphicsDevice.fullScreenWindow for reliable fullscreen detection
  - Add WindowFocusManager.getWindow() to access AWT windows by ID
  - Add BossLogger debug logging for the close sequence
  - No Thread.sleep() - requestToggleFullScreen initiates exit and disposeAllBrowsersBlocking handles synchronization
