# BOSS v8.12.17

**Release Date**: October 2025

## Summary

This release introduces optional automatic update checking, giving users control over background update notifications while maintaining easy access to manual update checks.

## Highlights

- User-controlled automatic update checking with toggle in Settings
- Configurable check interval with immediate effect on settings changes
- Improved "Last checked" time display with accurate relative timestamps
- Settings persist across app restarts

## Features

### Automatic Update Checking
- **Optional automatic checks** (#108)
  - Toggle in Settings → Updates to enable/disable automatic update checks
  - Default: Enabled (preserves existing 6-hour check behavior)
  - Immediate effect when toggled (no restart required)
  - Manual "Check for Updates" button always works regardless of toggle state

- **Configurable check interval** (#108)
  - checkIntervalHours setting now actually used (previously ignored)
  - Default interval: 6 hours
  - Stored in ~/.boss/update-settings.json for future UI control
  - Affects both periodic checks and shouldCheckForUpdates() logic

### Cross-Platform Settings Management
- **UpdateSettingsManager** (#108)
  - Follows existing TerminalSettingsManager pattern
  - expect/actual pattern for cross-platform compatibility
  - JSON persistence to ~/.boss/update-settings.json
  - Immediate loading and saving on settings changes

## Improvements

### User Experience
- **Accurate time display** (#108)
  - Fixed "Last checked" to show correct relative time
  - Better time granularity: minutes, hours, days, weeks, months, years
  - Proper singular/plural handling ("1 week" vs "2 weeks")
  - Comprehensive time formatting with Clock.System.now() - instant duration

## Bug Fixes

### Update Settings
- **Fixed "Last checked" calculation** (#108)
  - Previously showed incorrect times like "20386 days ago" for recent checks
  - Was converting absolute timestamp to days since Unix epoch instead of duration
  - Now correctly calculates duration between now and last check time
  - Shows accurate relative times: "Just now", "2 hours ago", "3 days ago", etc.

- **Fixed hardcoded check interval** (#108)
  - Removed unused CHECK_INTERVAL_HOURS constant
  - UpdateManager now uses UpdateSettings.checkIntervalHours
  - Setting was previously defined but never actually read

### Build System
- **Fixed missing imports** (#108)
  - Added missing kotlin.time.Clock import in UpdateUI.kt
  - Resolved compilation errors for Clock, inWholeDays, inWholeHours, inWholeMinutes
