# BOSS v8.17.20

**Release Date:** 2026-03-08

Microkernel architecture foundation enabling OS-like plugin isolation with gRPC-based IPC, AI-powered self-healing orchestration, and Supabase reliability fixes for secret manager stability.

## Highlights

- **Microkernel architecture** (gated by `ProcessMode`): Plugins and services can run in isolated processes with gRPC-based IPC, DAG workflow composition via `boss-mastery`, and self-healing orchestration — `MONOLITH` mode preserves all existing behavior
- **AI-powered repair**: `RepairEngine` with an escalation ladder calls an OpenAI-compatible LLM (via `AI_REPAIR_API_KEY`) to propose fixes for crashed processes; graceful no-op when the key is absent
- **Supabase Realtime stability**: Websocket heartbeat timeout increased to 30s with exponential-backoff reconnection (5s–60s), preventing secret manager hangs caused by a crashed Realtime channel destabilizing the shared Ktor engine
- **Security and code-quality hardening**: Path-traversal guards on all file service endpoints, process ID validation, Unix socket files secured at `0700`, `@Volatile` on all 12 IPC event bridge vars, and TCP port-race fix on Windows

## Features

- **kernel**: Add microkernel foundation — `boss-ipc`, `boss-process-manager`, `boss-orchestrator`, `boss-ui-sdk`, `boss-service-auth`, `boss-plugin-runtime`, `boss-mastery-sdk`, `boss-mastery-orchestrator`, `plugins/plugin-api-ipc` (10 new modules, all gated by `ProcessMode.KERNEL`)
- **kernel**: `PluginManifest` extended with `capabilities`, `repairHints`, `behaviorSpec`, `isolationMode`, `healthContract`, `sourceFiles` (quine self-description for AI repair)
- **kernel**: `IpcEventBridgeImpl` wired to all 12 event buses for cross-process event forwarding
- **auth**: `SupabaseAuthClient` + `AuthServiceGrpcImpl` — real sign-in/sign-out via Supabase SDK with session restore on startup
- **orchestrator**: `AiRepairClient` provider-agnostic interface; `HttpAiRepairClient` using `java.net.HttpURLConnection` (no extra deps); activated via `AI_REPAIR_API_KEY` env var

## Bug Fixes

- **supabase**: Fix `PluginStoreRealtimeService` websocket heartbeat timeout crashing the secret manager — increase heartbeat to 30s and add exponential-backoff reconnection loop with proper `CancellationException` handling
- **supabase**: Fix indefinite hangs in Postgrest requests (e.g. `get_user_secrets_with_shared`) caused by unconfigured CIO engine — set 30s request timeout, 15s connect timeout (1 retry), 30s socket idle, 20 max connections per route
- **kernel**: Fix TCP port race on Windows — cache allocated ports per `processId`
- **kernel**: Fix race condition in `KernelServiceImpl` heartbeat map — use `AtomicReference`
- **kernel**: Fix orchestrator `ProcessRegistry` permanently empty — `RepairEngine` now reads manifest from crash report directly and invokes restart callback to kernel
- **kernel**: Fix `ChildProcessBootstrap` throwing at construction — convert eager properties to lazy
- **kernel**: Fix approved repairs never executed — add `onRepairApproved` callback
- **kernel**: Fix `ProcessMonitor.monitorJobs` thread safety — use `ConcurrentHashMap`
- **kernel**: Fix `EventBusServiceImpl` double subscriber count — separate `anonIdCounter`
- **kernel**: Fix `StateServiceImpl` version atomicity — `Mutex`-guarded updates
- **kernel**: Fix `AuthDataProviderProxy` reconnection — `while(isActive)` + backoff loop
- **kernel**: Fix `MasteryServiceImpl` job race — register before suspension
- **kernel**: Fix `ProcessSpawner.findJavaExecutable()` to handle macOS app bundles
- **browser**: `BrowserServiceImpl.getPageInfo` warns on multi-window ambiguity instead of returning stale state

## Improvements

- **kernel**: `KernelBootstrap` spawns services via `spawnIfJarExists()` and uses `waitFor()` (not `Thread.sleep()`) during shutdown
- **kernel**: Unix socket files secured with POSIX `0700` permissions (SEC1)
- **kernel**: Path-traversal guard added to `FileSystemServiceImpl` and `EditorServiceImpl`
- **kernel**: `processId` validated against `[a-zA-Z0-9._-]+` before being used in socket paths
- **kernel**: `HttpAiRepairClient.callApi` wrapped in `withContext(Dispatchers.IO)`
- **kernel**: `println` replaced with `SLF4J logger.warn` in `IpcAddressResolver`
- **kernel**: `@Volatile` added to all 12 event bus `ipcBridge` variables
- **kernel**: GraalVM `metadataRepository` comment added to all 10 new module build files
- **kernel**: 100ms startup delay in `ChildProcessBootstrap.connect()` to avoid connect-before-bind races
- **orchestrator**: `OutOfProcessPluginSandbox` with callback-based lifecycle and local health tracking
- **services**: Full implementations for `TerminalServiceImpl`, `EditorServiceImpl`, `BrowserServiceImpl`, `WorkspaceServiceImpl`, `SettingsServiceImpl`, `FileSystemServiceImpl` (no more stubs)
