# BOSS v9.2.37

**Release Date:** 2026-07-09

Fixes the plugin-platform API hot swap: force-unload now reaches the loader so the four `canUnload=false` system plugins reload cleanly, and plugin UI is torn down before the API-layer swap closes classloaders â€” eliminating the crash dialogs a swap could trigger (#848).

## Highlights

- **API hot swap reloads all plugins cleanly**: `uninstallPlugin(force = true)` now flows through to the loader's `unloadPlugin`, which previously re-checked `canUnload` unconditionally. That left the four `canUnload=false` system plugins (plugin-manager, terminal-tab, editor-tab, fluck-browser) parented to the closed old `ApiClassLoader` and failing reload as "already loaded". A live swap (1.0.63â†’1.0.64) now cycles all 29 plugins in ~1.5s instead of leaving 4 stranded (#848).
- **No more crash dialogs during a swap**: The hot swap used to close plugin classloaders while their Compose UI was still mounted, so the terminal tab's `onDispose` ran against a closed loader and hit `NoClassDefFoundError`. `hotSwapApiLayer` now invokes a UI-teardown hook to close every plugin tab across all windows on the EDT before the unload loop, so disposal resolves against still-open loaders â€” zero uncaught exceptions where there were previously two `CrashHandler` dialogs (#848).

## Bug Fixes

- **plugins**: force-unload now reaches the loader during the API hot swap â€” `uninstallPlugin(force = true)` flows through to `unloadPlugin` (default `force = false`, so normal unload paths are unchanged), so the four `canUnload=false` system plugins no longer survive the unload with classloaders parented to the closed old `ApiClassLoader` and fail reload as "already loaded" (#848)
- **plugins**: plugin UI is torn down before the API-layer swap closes loaders â€” `hotSwapApiLayer` invokes `PluginLoaderDelegateImpl.teardownAllPluginTabs` to close every plugin tab across all windows on the EDT (and wait) before the unload loop, so Compose `onDispose` (e.g. BossTerm's terminal disposal) resolves against still-open loaders instead of raising `NoClassDefFoundError` and popping `CrashHandler` dialogs (#848)
